# Claude Code LP 更新マニュアル

> [!info] このマニュアルでできること
> `https://claudecode.n1-inc.co.jp/` のLP（ランディングページ）を、自分のPCで編集してインターネットに反映させる手順を、最初から最後まで通しで説明します。
> **対象**：HTML/サーバーの専門知識がない人。コマンド操作は使いません、すべて画面クリックで完結します。

---

## 0. 全体像（まずはココを把握）

LPは「**3つの場所**」を行き来して更新します。

```
[1] 自分のPC（ローカル）        [2] XServer（インターネット上のサーバー）        [3] ブラウザ
   HTMLファイルを編集    ──→    ファイルマネージャでアップロード    ──→    実際のLPに反映
```

ざっくり：
1. **自分のPCで** HTMLを書き換える
2. **XServerに** そのファイルをアップロードして上書き
3. **ブラウザで** 反映を確認する

それだけです。

## 2. 用語ミニ辞典（最初に読む）

専門用語が出てくるので、ここで意味を押さえておく。

| 用語 | ざっくり意味 |
|-----|-----------|
| **HTML** | Webページの中身を書く言語。テキストエディタで開ける普通のファイル |
| **サーバー** | インターネット上にWebページを置いておく「倉庫」のこと |
| **ホスティング** | その倉庫を借りて自分のWebページを置くこと |
| **ドメイン** | `n1-inc.co.jp` のような「住所」 |
| **サブドメイン** | `claudecode.n1-inc.co.jp` のように、メインドメインの前にくっつけた住所 |
| **ドキュメントルート** | サーバー上で「ここに置いたファイルがWebに公開される」と決められた場所 |
| **index.html** | URLにアクセスしたとき最初に表示される特別なファイル名 |
| **アップロード** | 自分のPC → サーバーへファイルを送ること |
| **ハード再読み込み** | ブラウザのキャッシュを無視して最新版を取りに行く操作 |

## 4. 更新手順（本編）

### STEP 1 — ローカルでHTMLを編集する

1. 自分のPCで、Obsidian Vaultのフォルダを開く
2. `marketing/claudecode_training_lp.html` をテキストエディタ（VS Code、メモ帳、Cursorなど何でも可）で開く
3. 修正したい箇所を書き換える
4. **上書き保存**する

> [!tip] 編集に自信がない場合
> Claudeに「LPのこの部分をこう変えたい」と頼めば、HTMLを直接修正してくれます。
> 例：「ヒーローのキャッチコピーを『〇〇〇』に変えて」

### STEP 2 — `index.html` という名前のコピーを作る

サーバーに置くときの名前は `index.html` で固定です。ローカル名（`claudecode_training_lp.html`）のままだと公開URLで表示されません。

1. `claudecode_training_lp.html` を**コピー**する（複製）
2. コピー後のファイルを `index.html` に**リネーム**する
3. デスクトップ等のわかりやすい場所に置いておく
   - 推奨：`~/Desktop/claudecode_lp_deploy/index.html`

> [!warning] よくあるミス
> 元のファイル（`claudecode_training_lp.html`）の名前を直接変えてしまうと、Vault内のソースファイルが消えてしまいます。**必ずコピーしてからリネーム**してください。

### STEP 3 — XServerにログインする

1. ブラウザで以下にアクセス
   👉 https://www.xserver.ne.jp/login_server.php
2. **サーバーID** に `xs842147` を入力
3. **サーバーパスワード** を入力
4. 「ログイン」ボタンをクリック

> [!note] ログイン画面の名前
> 「**Xserverアカウント**」と「**サーバーパネル**」の2種類があります。今回使うのは **サーバーパネル** のほうです。間違えてアカウント側に入った場合は、画面内のメニューから「サーバーパネル」に切り替えればOK。

### STEP 4 — ファイルマネージャを開く

1. サーバーパネルのトップ画面で「**ファイル**」というセクションを探す
2. その中の「**ファイルマネージャ**」をクリック
3. 新しいタブが開き、サーバー内のフォルダ一覧が表示される

### STEP 5 — 公開フォルダまで降りていく

ファイルマネージャの左側または上側のフォルダ階層を、以下の順にダブルクリックで進む。

```
n1-inc.co.jp
  └─ public_html
       └─ claudecode.n1-inc.co.jp     ← ここがゴール
```

たどり着くと、画面に既存の `index.html` と `assets` フォルダが見えるはず。

> [!warning] 似た名前のフォルダに注意
> `public_html` の直下には `n1-inc.co.jp` 本体のファイルも並んでいます。**必ず `claudecode.n1-inc.co.jp/` の中**で作業してください。違うフォルダに上書きすると、コーポレートサイトが壊れる可能性があります。

### STEP 6 — 既存のindex.htmlをバックアップ（保険）

万一に備えて、今のindex.htmlを取っておく。

1. 既存の `index.html` を右クリック
2. 「**ダウンロード**」を選択
3. 自分のPCに `index.html` が落ちてくる
4. 名前を `index_backup_YYYYMMDD.html` に変えてデスクトップに保管

> [!tip] なぜバックアップ？
> もし新しいファイルでLPが崩れたとき、このバックアップを再アップロードすれば**1分で元に戻せる**から。必ずやる。

### STEP 7 — 新しいindex.htmlをアップロード

1. ファイルマネージャの上部メニューから「**ファイルのアップロード**」をクリック
2. ダイアログが開いたら、ローカルで作った `index.html`（STEP 2のもの）を選択
3. 「アップロード」ボタンをクリック
4. 「**同名のファイルを上書きしますか？**」と聞かれたら **「はい」** を選ぶ
5. アップロード完了のメッセージが出るまで待つ（数秒〜十数秒）

> [!success] アップロード成功の見分け方
> ファイル一覧の `index.html` の「更新日時」が、たった今の日時に変わっていればOK。

### STEP 8 — ブラウザで反映を確認する

1. 新しいタブを開く
2. アドレスバーに `https://claudecode.n1-inc.co.jp/` を入力してEnter
3. **ハード再読み込み**で最新版を取得する

> [!info] ハード再読み込みのキー操作
> - **Mac**：`Cmd + Shift + R`
> - **Windows**：`Ctrl + Shift + R` または `Ctrl + F5`
>
> 普通のリロード（`Cmd + R` / `F5`）だと、ブラウザが古いキャッシュを表示してしまい「変わってない！」と勘違いするので必ずハード再読み込み。

4. 修正した箇所がちゃんと反映されているかを目視で確認
5. PC・スマホ両方で見るとなおよし

---

## 5. ロゴ画像など、画像を差し替えたいとき

手順はHTMLとほぼ同じ。違うのは置き場所だけ。

1. 新しい画像を `n1-logo.jpg` という同じ名前にリネーム
2. ファイルマネージャで `claudecode.n1-inc.co.jp/assets/` フォルダに入る
3. STEP 7と同じ要領でアップロード（上書き）
4. ブラウザでハード再読み込み

> [!warning] ファイル名は変えない
> HTMLの中で `assets/n1-logo.jpg` という名前で画像を呼び出しています。ファイル名を変えるとリンクが切れて画像が表示されなくなります。**中身だけ差し替え、名前は据え置き**が原則。

## 6. トラブルシューティング

### 「変更が反映されない」

- [ ] ハード再読み込みをやったか？（`Cmd+Shift+R`）
- [ ] アップロード先のフォルダは `claudecode.n1-inc.co.jp/` で合っていたか？
- [ ] ファイル名は `index.html`（小文字・拡張子.html）か？ `Index.html` や `index.HTML` はNG
- [ ] スマホで見ても同じか？（PCのキャッシュ問題切り分け）

### 「ページが真っ白／崩れている」

すぐにバックアップを戻す。
1. STEP 6で取っておいた `index_backup_YYYYMMDD.html` を `index.html` にリネーム
2. STEP 7と同じ手順でアップロード（上書き）
3. ハード再読み込みで元に戻ったか確認

### 「ログインできない」

- サーバーIDは `xs842147`（数字を打ち間違えやすい）
- パスワードは1Password等で再確認
- それでもダメなら大川（r.okawa@n1-inc.co.jp）に連絡

### 「ファイルマネージャで上書きの確認ダイアログが出ない」

XServer側のUI仕様で、出ない場合もあります。アップロード後に必ず**更新日時を確認**して、上書きが成功しているかをチェックしてください。

## 7. やってはいけないこと（重要）

> [!danger] NGリスト
> - ❌ `claudecode.n1-inc.co.jp/` 以外のフォルダにアップロードする（コーポレートサイトに影響）
> - ❌ `index.html` 以外の名前でアップする（公開URLに表示されない）
> - ❌ バックアップを取らずに上書きする（戻せなくなる）
> - ❌ XServerのログイン情報をメール／Slackで平文共有する（漏洩リスク）
> - ❌ サーバーパネル内の「ドメイン」「SSL」設定をいじる（公開そのものが止まる）
