## 本文

業務自動化SaaSツール「Kawaru（カワル）」の開発・提供を行う株式会社エヌイチ（本社：東京都新宿区、代表取締役CEO：奥山幸生、以下「エヌイチ」）は、チャットで話しかけるだけで誰でも業務の自動化フローを構築できるSaaSツール「Kawaru」を正式リリースしたことを発表しました。個人のビジネスパーソンから大企業の現場チームまで幅広く活用できる設計となっており、Salesforce・HubSpot・GitHub・Notionなど業界を代表するSaaSツールとの連携にも対応。あらゆる業務の自動化を、専門知識ゼロで実現します。


### 「AIを導入したのに、誰も使っていない」——85%の企業が直面する現実

生成AIへの関心が高まる一方、日本企業の生成AI活用状況は「組織的に活用できている企業はわずか15%」にとどまっています（エヌイチ社調べ、1,000社以上へのヒアリングより）。残る85%の企業では「試したが定着しない」「何から始めればいいかわからない」という状態が続いています。

定着しない理由として、現場からは以下の声が多く挙がっています。

- 「ワークフロー自動化はエンジニアがいないと作れないと思っていた」
- 「AIツールを入れたが、現場の業務に合っておらず使われなくなった」
- 「自動化したい業務はあるが、専門知識がなく手が出せない」

また、1,000社以上の現場担当者へのヒアリングでは、**ビジネスパーソンの約80%がメール対応・議事録作成・データ入力などの「雑務・定型業務」に時間を奪われており、本来の業務に使えている時間はわずか20%** という実態も明らかになっています。


### Kawaru（カワル）とは

Kawaruは、「チャットで話しかけるだけで、誰でも業務の自動化フローがつくれる」SaaSツールです。個人のフリーランス・スタートアップから、数百名規模の組織まで、あらゆるビジネスパーソンが初日から使える設計になっています。

従来、業務の自動化フロー（ワークフロー）の構築には、エンジニアリングの知識やプロンプトエンジニアリングのスキルが不可欠でした。結果として「業務を知っている現場の担当者」と「ツールを作れる専門人材」が分断され、"現場の実態に合わない自動化"が量産されてきました。

Kawaruは、この構造を根本から変えます。日本語で「やりたいこと」を伝えるだけで、AIがヒアリングしながら最適なフローを自動で構築。現場を一番知っている担当者が、自分で自動化を作れる環境を実現しました。


### Kawaruの3つの特徴

**（1）専門知識ゼロで、初日から使える**

エンジニアリング知識・AI知識は一切不要です。「このメール対応を自動化したい」「会議が終わったら議事録をSlackに投稿したい」——やりたいことを日本語で伝えるだけで、AIが最適なワークフローを自動構築します。ノードをつなぐ作業も、プロンプトの設計も必要ありません。

**（2）個人の効率化を、組織全体に広げられる**

自分が作ったワークフローを、チーム全体のテンプレートとしてワンクリックで展開できます。現場の担当者一人が作った自動化が、部署全体・組織全体へと自然に波及する仕組みを持っています。個人の工夫を「組織の資産」に変えることが可能です。

**（3）既存ツールと連携し、メンテナンスフリーで動き続ける**

Slack・Chatwork・Gmail・Zoomなど、すでに使っている社内ツールとシームレスに連携。RPAのようにUI変更で止まることなく、自己修復・自動更新で動き続けます。導入後も専任CSスタッフが継続サポートするため、「使い始めたはいいが止まったまま」という状況を防ぎます。


### 活用イメージ——「チャット1つ」で実現できること

Kawaruで構築できるワークフローの例は以下の通りです。

**【営業・CRM】商談情報のSalesforce自動入力**
商談完了 → 会話内容を自動取得 → データ整形 → Salesforceへ自動登録。営業担当者の手入力工数をゼロにします。見込み客の行動データをSales Markerから取得し、Salesforceのリードステータスを自動更新するといった複数ツール連携も、チャット1つで構築できます。

**【マーケティング】MAツール連動による見込み客ナーチャリング自動化**
HubSpotやMarketo等のMAツールと連携し、フォーム入力 → スコアリング → メール配信 → 担当者通知までを自動化。これまで専任のマーケターが手動で管理していたリードの育成プロセスを、ノーコードで再現します。

**【エンジニアリング】GitHubと連携したレビュー・通知フロー**
GitHubでプルリクエストが作成されると、担当者をSlackへ自動通知。レビュー完了後にNotionの進捗ボードを更新し、Jiraのチケットステータスも連動して変更。開発チームのコミュニケーションコストを大幅に削減します。

**【採用・HR】候補者情報の一元管理と選考フロー自動化**
応募情報の受信 → 候補者データの自動整形 → HRシステムへの登録 → 担当採用担当者へSlack通知。面接日程の調整からGoogleカレンダーへの自動登録まで、採用担当者の事務工数を極限まで削減します。

**【会議・議事録】Slack自動投稿と関係者への展開**
会議終了 → 文字起こし自動取得 → AIが議事録生成 → Slackへ投稿 → Notionの議事録データベースへ自動保存。従来60分かかっていた作業が0秒で完了。決定事項はそのまま各担当者のタスク管理ツールへ自動連携されます。

**【メール】受信から返信下書きまで全自動**
メール受信 → AI要約・文脈学習 → 返信文生成 → 下書き保存。1通あたり数分かかっていた対応が、受信と同時に完了します。

これらはすべて、チャットで「やりたいこと」を伝えるだけで構築されます。


### 対応ツール——あらゆる業務環境に接続できる

Kawaruは、企業が日常的に使用する主要SaaSツールとの連携に対応しています。

| カテゴリ | 対応ツール（一例） |
|---------|-----------------|
| コミュニケーション | Slack、Chatwork、Microsoft Teams、Gmail |
| CRM・営業支援 | Salesforce、HubSpot、Sales Marker |
| プロジェクト管理 | Notion、Jira、Asana、GitHub |
| MAツール | HubSpot、Marketo、Pardot |
| HR・採用 | AWAMO、各種ATSシステム |
| 会議・録画 | Google Meet、Zoom、Microsoft Teams |
| ストレージ | Google Drive、OneDrive、Dropbox |

連携ツールは順次拡大予定。自社で利用しているツールがリストにない場合も、API連携による対応が可能です。


### 既存ツールとの違い

KawaruはRPAや既存のワークフローツール（Zapier等）と比較して、以下の点で根本的に異なるアプローチを取っています。

RPAや既存ワークフローツールは定型作業のみへの対応・UI変更による停止・専門知識が必要という課題がありました。Kawaruは「判断を伴う業務」にも対応し、自然言語で誰でも構築でき、自己修復によりメンテナンスフリーで動き続けます。


### 代表取締役CEO 奥山幸生 コメント

「業務の自動化は、一部の専門人材だけのものであってはならないと考えています。現場の担当者が自分の言葉で自動化を作れる環境こそが、組織全体の生産性を底上げする唯一の方法です。Kawaruは、AIで"変わる"のは業務だけでなく、組織そのものだという考えのもと開発しました。まず現場から、一つの自動化から始めてください」


### 導入の流れ

Kawaruの導入は3つのステップで完了します。

**Step 1：ヒアリング**
専任CSスタッフが御社の課題・業務フローをヒアリング。「どこから自動化すべきか」の特定から支援します。「何から始めればいいかわからない」という段階からご相談いただけます。

**Step 2：30分で初期構築**
アカウント発行・既存ツール（Slack / Chatwork等）との連携設定・初期ワークフロー構築をすべてCSスタッフが代行。御社側の作業負担はほとんどありません。

**Step 3：現場で利用開始**
あとは使うだけ。AIの知識は一切不要です。導入後も専任CSスタッフが継続サポートします。
