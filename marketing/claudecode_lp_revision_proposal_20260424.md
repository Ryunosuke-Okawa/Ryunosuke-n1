# Claude Code LP 改修提案

## 背景

現状のLPは「Claude Code研修＋コンサル」のシンプルな見せ方になっているが、**N1の強み**が見えない。
ベースとなっているKawaru Team（研修）／Kawaru Coach（顧問）の強みをClaude Code文脈に置き換えて反映したい。

## 現状の弱点

| 観点 | 現状 | 課題 |
|------|------|------|
| なぜN1なのか | 訴求なし | 競合（デジライズ等）との差別化が見えない |
| 研修の中身 | 4項目の箇条書き | 「具体的に何が違うのか」が薄い |
| コンサルの中身 | 3フェーズ（戦略・推進・定着） | Kawaru Coachの強みである「4,000名のAIコンサル」「経営×事業×組織の3視点」が出ていない |
| 課題提起 | 3つで一般論 | Kawaru Team/Coach資料にある具体課題（「研修受けたのに使われない」「外部コンサルが提案だけで終わる」）が刺さる |

---

## 強みマッピング（既存サービス → Claude Code版）

### Kawaru Team の強み → Claude Code研修に転用

| Kawaru Team | Claude Code研修版 |
|------------|-----------------|
| 完全フルカスタマイズ（パッケージなし） | **貴社のコードベース・業務に合わせ毎回ゼロから設計** |
| 組織変革を設計（研修して終わりにしない） | **研修後の運用ルール整備・内製化まで一気通貫** |
| 楽しくなければ続かない体験型 | **講義より、翌日使える「できた」体験を重視** |
| 3ステップ研修（基礎→実務→実践） | **3ステップ（基礎→Claude Code実機→チーム実践）** |
| キリン業務活用率100%・ミズカラ283→85h | （実績は今回非表示方針なので削除） |

### Kawaru Coach の強み → Claude Codeコンサルに転用

| Kawaru Coach | Claude Codeコンサル版 |
|-------------|-------------------|
| 4,000名超のAIコンサルタント | **4,000名超のAIプロ人材プールから貴社最適の担当をアサイン** |
| 経営・事業・組織の3視点 | **技術支援にとどまらず、事業・組織視点まで踏み込む** |
| マンツーマン専属顧問 | **社内に1人「Claude Code＋AIプロ」を持つ感覚** |
| 戦略言語化〜実装まで一気通貫 | **「提案で終わる外部コンサル」とは違い、現場実装まで伴走** |
| 3フェーズ（言語化→伴走→定着横展開） | **同じ3フェーズ構成** |
| 月額制で継続・解約自由 | **同様の契約設計（成果コミット）** |

## 提案する3つの改修

### 【改修1】新セクション「Why N1」を追加

**配置候補**：Programsの直前 or Processの後

**コンセプト**：「なぜClaude Code法人支援を、N1に頼むべきか」を4つの理由で示す

#### 草案

> ## なぜ、エヌイチなのか
>
> Claude Codeを扱える会社は他にもある。<br>
> しかし、組織に根づかせられる会社は限られる。

| # | 強み | サブコピー |
|---|------|---------|
| 01 | **パッケージは、ひとつもない** | 業種・組織規模・ITリテラシーが同じ会社は存在しない。だから貴社専用にゼロから設計する。 |
| 02 | **4,000名超のAIプロ人材プール** | AIスクール事業で育てた4,000名超のAIコンサルタントから、貴社の業種・課題に最適な担当者をアサイン。 |
| 03 | **経営・事業・組織の3視点で支援** | Claude Codeの使い方を教えるだけではない。事業目標・組織設計まで踏み込んで「実装」まで伴走する。 |
| 04 | **「終わったら終わり」にしない** | 研修・コンサルが終わった後も、組織にAI活用が文化として根づくまで内製化を支援する。 |

→ アイコン付き4カードで配置（既存`.value-card`スタイル流用可）

---

### 【改修2】Programsセクションをリッチ化

#### Training（研修）カード

**現状**：特徴4項目の箇条書き

**改修案**：3ステップ構成に変更（Kawaru Teamの「研修の3ステップ」転用）

```
STEP 01  Claude Codeを「自分ごと」で理解する
業務題材に置き換えた講義設計。「ツールの使い方」ではなく
「あなたの仕事がどう変わるか」にフォーカス。

STEP 02  実務でClaude Codeを動かす
貴社のコードベース・業務題材を使ったハンズオン。
半日で「翌日から使える」状態に。

STEP 03  チームで成果を出すワークショップ
実務課題をテーマに、参加者がClaude Codeで成果物を作る。
研修終了時点で「使える成果物」が残る完全実践型。
```

カード下部の特徴箇条書きは、3〜4項目に絞って残す：
- パッケージプランは持たない（フルカスタマイズ）
- エンジニア／非エンジニア別クラス対応
- 集合・オンライン・ハイブリッド対応
- 半日WS〜複数回シリーズまで

#### Consulting（コンサル）カード

**現状**：3フェーズロードマップ（戦略設計／推進体制／運用定着）

**改修案**：Kawaru Coachの3フェーズに完全寄せ

```
PHASE 01  「何をすべきか」を言語化する
経営目標・組織体制から、Claude Codeで取り組むべき
優先課題を明文化。納得しなければ次に進まない。

PHASE 02  AI顧問が隣で動き、実装を前に進める
月2〜4回のMTG。Claude Code導入支援・進捗管理・壁打ちを継続。
「計画が止まる」を起こさない体制を作る。

PHASE 03  成功体験を組織全体に広げる
個別の改善成果を社内の横展開・ナレッジ化・AI人材育成へ。
外部依存から脱却し、文化として根づく状態へ。
```

特徴箇条書き：
- 4,000名超のAIプロ人材から最適な担当をアサイン
- 経営・事業・組織の3視点で伴走
- 月額制で継続。3ヶ月以降は1ヶ月単位で解約可
- MTG外でもSlack・メールで相談対応

---

### 【改修3】Challengesセクションをシャープ化

**現状**：一般論3カード

**改修案**：Kawaru Team / Coach資料の「解決できる課題」から、Claude Code文脈で刺さる4つに絞る

```
01  Claude Code研修を社内向けに考えたが、市販パッケージが業務に合わない
02  外部のAIコンサルに頼んだが、提案書だけで実装まで進まない
03  研修を実施したが、現場で使われずにフェードアウト
04  社内の一部エンジニアしか使えず、組織全体の成果に繋がらない
```

または、現状の3カードはそのまま残し、新しい1カードを追加して4カードにする選択も可能。

## 改修後の全体構成（案）

```
1. Header
2. Hero（変更なし）
3. Meta Strip（変更なし）
4. Challenges ← シャープ化（4カードに）
5. Why N1 ← 新規追加（4つの強み）
6. Programs ← Training/Consultingカードをリッチ化
7. Process（変更なし）
8. FAQ（変更なし）
9. Contact（変更なし）
10. Footer
```

## 進め方の提案

### Option A：3つすべてを一気に実装
- 工数：1セッションで完結
- 反映：再アップロード1回

### Option B：段階的に実装
- まず「Why N1」セクションだけ追加 → プレビュー → 次にProgramsリッチ化 → ...
- 細かく調整しながら進められる

### Option C：一部だけ採用
- 例：「Why N1」だけ追加、Programsは現状維持
- 最小コストで強み訴求を追加
