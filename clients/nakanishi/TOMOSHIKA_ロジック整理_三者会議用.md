# TOMOSHIKA ロジック整理（三者会議用）

> 適性検査の回答から企業推薦結果が表示されるまで、データがどう流れ・どう処理されるかを4フェーズで整理したもの。

---

## 全体フロー

```
┌──────────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐
│ フェーズ1  │      │ フェーズ2  │      │ フェーズ3  │      │ フェーズ4  │
│ 適性検査   │ ──→ │ MIL→N1   │ ──→ │ N1内判定  │ ──→ │ N1→MIL   │
│ →タイプ分類│      │ データ送信 │      │ ロジック   │      │ データ返却 │
│【MIL側】  │      │【連携部分】│      │【N1/Dify】│      │【連携部分】│
└──────────┘      └──────────┘      └──────────┘      └──────────┘
    学生が               JSON で            4段階判定で          JSON で
   51問に回答          スコアを送信        企業をマッチング      結果を返却
```

## フェーズ1: 適性検査 → タイプ分類【MIL側】

### 質問票の構造（全51問）

| 設問 | 分類 | 内容 | 用途 |
|------|------|------|------|
| Q1〜Q24 | RIASEC | 各タイプ4問 × 6タイプ（5点満点 → 各20点満点） | 志向性タイプの判定 |
| Q25〜Q36 | ポータブルスキル | ビッグファイブ応用（12問） | コンピテンシー評価 |
| Q37〜Q48 | キャリア・アンカー | シャイン理論（12問） | 価値観・カルチャーマッチ |
| Q49 | L尺度 | 虚偽回答検知（1問） | スコア補正用 |
| Q50 | BtoB志向 | BtoB / BtoC の志向（1問） | 企業リスト切替用 |
| Q51 | インフラ/メーカー志向 | 中西親和性判定（1問） | 中西該当判定用 |

### 6タイプ（RIASEC）

| コード | タイプ名 | 特徴 |
|--------|---------|------|
| R | 現実型 | ものづくり・技術志向 |
| I | 研究型 | 分析・探究志向 |
| A | 芸術型 | 創造・表現志向 |
| S | 社会型 | 対人支援・教育志向 |
| E | 企業型 | リーダーシップ・経営志向 |
| C | 慣習型 | 管理・正確性志向 |

### このフェーズで出るもの

学生が51問に回答し、MIL側で以下のスコアが算出される：
- **R, I, A, S, E, C**: 各タイプの合計点（各0〜20点）
- **Q30, Q31**: 行動特性に関する個別設問の回答値
- **Q49**: L尺度（虚偽回答検知）
- **Q50**: BtoB志向スコア
- **Q51**: インフラ/メーカー志向スコア

> [!question] 会議で確認すべきポイント
> - RIASECの合計点算出はMIL側で行うのか？（生の回答データをそのまま送るのか、集計済みスコアを送るのか）
> - L尺度補正（Q49による × 0.8）はどちら側で処理するか？

## フェーズ2: MIL → N1 へのデータ送信【連携部分】

### 送信方法

MILのサーバーから、N1のDify APIに対してHTTPリクエスト（JSON形式）で送信。

### 送信データ（想定）

```json
{
  "R":  15,    // R型スコア（0〜20）
  "I":  12,    // I型スコア（0〜20）
  "A":   8,    // A型スコア（0〜20）
  "S":  18,    // S型スコア（0〜20）
  "E":  10,    // E型スコア（0〜20）
  "C":   6,    // C型スコア（0〜20）
  "Q30":  4,   // 行動特性①
  "Q31":  4,   // 行動特性②
  "Q49":  3,   // L尺度（虚偽回答検知）
  "Q50":  4,   // BtoB志向
  "Q51":  4    // インフラ/メーカー志向
}
```

全11項目、すべてNumber型。

### 認証方式

- Dify APIキーによる認証（`Authorization: Bearer sk-xxx`）
- 現時点はクラウド版（WebApp無効・API only）で運用予定

> [!question] 会議で確認すべきポイント
> - JSONのキー名は上記で問題ないか（MIL側の実装に合わせる）
> - 各スコアの値域（0〜20? 0〜5?）の認識合わせ
> - エンドポイントURLの共有タイミング

## フェーズ3: N1内の判定ロジック【N1 / Dify】

ここが今回の**最重要パート**。受け取った11項目のスコアから、4段階の優先順位で判定を行う。

### 判定フロー全体図

```
入力: R, I, A, S, E, C, Q30, Q31, Q49, Q50, Q51
                    │
                    ▼
        ┌─────────────────────┐
        │  Step 0: L尺度補正   │
        │  Q49 == 5 の場合     │
        │  → RIASEC各スコア×0.8│
        └──────────┬──────────┘
                    │（補正後のスコアで以降を判定）
                    ▼
        ┌─────────────────────┐
        │  Step 1: 未分化判定   │  ← 最優先
        │  max - min ≦ 1.0 ?  │
        └───┬─────────┬───────┘
          Yes│         │No
            ▼          ▼
     【トモキャリ誘導】  │
      result: tomocari  │
                        ▼
        ┌─────────────────────┐
        │  Step 2: 行動特性判定 │
        │  Q30≧4 AND Q31≧4 ? │
        └───┬─────────┬───────┘
          Yes│         │No
            ▼          ▼
     【中西製作所＋      │
      同系統企業】       │
      result: nakanishi  │
                        ▼
        ┌─────────────────────────────┐
        │  Step 3: タイプ×志向判定      │
        │  上位2タイプにE/I/Aを含む     │
        │  AND (Q50+Q51)/2 ≧ 3.0 ?   │
        └───┬─────────────────┬───────┘
          Yes│                 │No
            ▼                  ▼
     【中西製作所＋       ┌──────────────────┐
      同系統企業】        │  Step 4: 通常判定  │
      result: nakanishi   │  1位タイプ×Q50で  │
                          │  メイン企業決定    │
                          │  2位タイプ×Q50で  │
                          │  補足企業決定     │
                          └────────┬─────────┘
                                   ▼
                            【タイプ別企業推薦】
                             result: type_match
```

### 各Stepの詳細

#### Step 0: L尺度補正

| 項目 | 内容 |
|------|------|
| 発動条件 | Q49 = 5（最大値）のときのみ |
| 補正対象 | RIASECスコア（R, I, A, S, E, C）のみ |
| 補正方法 | 各スコア × 0.8 |
| 例 | E = 20点 → 16点として以降の判定に使用 |
| 目的 | 社会的望ましさ（嘘回答）の傾向がある場合にスコアを抑制 |

#### Step 1: 未分化判定（最優先）

| 項目 | 内容 |
|------|------|
| 条件 | max(R,I,A,S,E,C) − min(R,I,A,S,E,C) ≦ 1.0 |
| 意味 | 6タイプのスコアに差がほぼない → 志向が定まっていない |
| 結果 | **トモキャリ誘導**（特定企業を推薦せず、キャリア相談へ） |

#### Step 2: 行動特性判定

| 項目 | 内容 |
|------|------|
| 条件 | Q30 ≧ 4 **AND** Q31 ≧ 4（両方満たす） |
| 意味 | 主体性・変革意欲が高い → 中西製作所の社風に合致 |
| 結果 | **中西製作所を筆頭 ＋ 同系統企業**（キーエンス, 平田機工, マクニカ, ダイフク等） |

#### Step 3: タイプ × 志向判定

| 項目 | 内容 |
|------|------|
| 条件① | 1位 or 2位タイプのいずれかが **E（企業型）, I（研究型）, A（芸術型）** |
| 条件② | (Q50 + Q51) / 2 ≧ 3.0 |
| 両方を満たす | **中西製作所を筆頭 ＋ 同系統企業** |
| 意味 | 中西製作所と親和性の高い志向タイプ × BtoB/メーカー志向 |

#### Step 4: 通常判定（上記いずれにも該当しない場合）

| 項目 | 内容 |
|------|------|
| メイン企業 | **1位タイプ** × Q50（BtoB/BtoC）で企業グループを決定 |
| 補足企業 | **2位タイプ** × Q50（BtoB/BtoC）で企業グループを決定 |
| BtoB/BtoC切替 | Q50 ≧ 3.0 → BtoBリスト / Q50 < 3.0 → BtoCリスト |
| 表示 | メイン企業を主表示 ＋「〇〇の側面も」として補足企業を併記 |

### 企業マスタ（タイプ × BtoB/BtoC）

| タイプ | BtoB企業 | BtoC企業 |
|--------|---------|---------|
| 中西型 | 中西製作所, キーエンス, 平田機工, マクニカ, ダイフク | ソニー, リクルート, メルカリ, カヤック, マネーフォワード |
| R型 | 三菱電機, 村田製作所, 安川電機, 栗田工業, THK | トヨタ, 任天堂, スノーピーク, ヤマハ, マキタ |
| I型 | 野村総研, 島津製作所, 中外製薬, オムロン, ユーザーベース | Google, 武田薬品, コーセー, エーザイ, メルカリ |
| A型 | 電通, 博報堂, チームラボ, Sansan, 乃村工藝社 | サンリオ, オリエンタルランド, 資生堂, 集英社, 無印良品 |
| S型 | ベネッセ, リクルートマネジメント, パソナ, ワークスアプリ, LITALICO | 日本赤十字, 損保ジャパン, 星野リゾート, Zoff, クラシコム |
| E型 | 三菱商事, 伊藤忠, 光通信, サイバーエージェント, レバレジーズ | ソフトバンク, 楽天, ファーストリテイリング, ZOZO |
| C型 | 日本生命, 東京海上, 三菱UFJ, 富士通, オービック | JR東日本, ANA, 日本郵便, 公務員, ゆうちょ銀行 |

## フェーズ4: N1 → MIL へのデータ返却【連携部分】

### レスポンスJSON（想定）

```json
{
  "result_type": "nakanishi",
  "primary_type": "S",
  "secondary_type": "R",
  "primary_companies": "中西製作所, キーエンス, 平田機工, マクニカ, ダイフク",
  "primary_reasons": "（各企業の推薦理由テキスト）",
  "secondary_companies": "該当なし",
  "secondary_reasons": "該当なし",
  "message": "あなたの主体性と変革意欲は、安定基盤×ベンチャー志向の企業で大きく花開きます。"
}
```

### 各項目の意味

| 項目 | 内容 | 値の例 |
|------|------|--------|
| result_type | 判定カテゴリ | `nakanishi` / `type_match` / `tomocari` |
| primary_type | 1位の志向性タイプ | `R`, `I`, `A`, `S`, `E`, `C` |
| secondary_type | 2位の志向性タイプ | 同上 |
| primary_companies | 推薦企業リスト（メイン） | 企業名をカンマ区切り |
| primary_reasons | 推薦理由（メイン） | 各企業に対する理由テキスト |
| secondary_companies | 推薦企業リスト（補足） | 通常判定時のみ。中西該当時は「該当なし」 |
| secondary_reasons | 推薦理由（補足） | 同上 |
| message | 学生へのメッセージ | 判定結果に応じた一言 |

### result_type ごとの出し分け

| result_type | 意味 | primary | secondary |
|-------------|------|---------|-----------|
| `nakanishi` | 中西に適性あり（Step 2 or 3） | 中西＋同系統企業 | 該当なし |
| `type_match` | 通常のタイプ別マッチング（Step 4） | 1位タイプの企業 | 2位タイプの企業 |
| `tomocari` | 志向未分化（Step 1） | トモキャリ誘導 | 該当なし |

> - companies を配列にするか、カンマ区切り文字列にするか
> - MIL側での結果画面の表示仕様との整合
