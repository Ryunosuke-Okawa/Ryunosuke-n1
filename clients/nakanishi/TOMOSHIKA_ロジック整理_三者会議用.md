# TOMOSHIKA ロジック整理（三者会議用）

学生が適性検査を受けてから、結果画面に推薦企業が表示されるまでの流れを4つのフェーズに分けて整理したものです。

---

## 全体の流れ

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  フェーズ1     │    │  フェーズ2     │    │  フェーズ3     │    │  フェーズ4     │
│  適性検査      │ →  │  データを      │ →  │  企業の       │ →  │  結果を       │
│  を受ける      │    │  送る         │    │  マッチング    │    │  返す         │
│ 【MIL様】    │    │【MIL様→Dify】│    │ 【Dify】     │    │【Dify→MIL様】│
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
  学生が51問に        MIL様が検査結果を     受け取った点数で       マッチング結果を
  回答する            Difyに送る           企業リストと照合する    MIL様に返す
```

### 三社の役割

| 会社 | 何をするか |
|------|----------|
| **MIL様** | 学生に適性検査を受けてもらう画面を作る。検査結果をDifyに送り、返ってきた結果を学生に見せる |
| **エヌイチ** | 中西製作所様のDifyアカウント上に「検査結果を受け取って企業をマッチングするしくみ」を構築・運用する |
| **中西製作所様** | Difyアカウントの持ち主。どの企業を推薦するか、どういう条件で中西を出すかの最終決定権を持つ |

### Difyについて

- **Dify**は、プログラムの処理を組み立てて動かすためのツール（クラウドサービス）
- このDifyのアカウントは**中西製作所様のもの**
- エヌイチがこのアカウント上でしくみ（ワークフロー）を作っている
- フェーズ2・4で必要になる**接続先の情報（URL）や認証キー**も、中西製作所様のアカウントから発行される

---

## フェーズ1: 適性検査を受ける【MIL様側】

### このフェーズで何が起きるか

学生が51問の質問に答えると、その回答から**11個の数値（点数）**が算出される。この点数がフェーズ3の企業マッチングの材料になる。

### 質問の中身（全51問）

#### メインの質問: Q1〜Q24（志向性を測る24問）

この24問で**「この学生がどんな仕事に向いているか」を6タイプで数値化**する。

24問は6つのタイプに4問ずつ割り振られていて、各問は「1点（あてはまらない）〜 5点（あてはまる）」で答える。同じタイプの4問の合計がそのタイプの点数になる。

```
例: R型（現実型）の4問にそれぞれ 5点, 4点, 3点, 3点 と答えた場合
→ R型スコア = 5 + 4 + 3 + 3 = 15点（20点満点中）
```

これを6タイプ分計算する:

| タイプ | 対応する設問 | 算出方法 | 点数の範囲 |
|--------|-----------|---------|-----------|
| **R型**（現実型） | Q1〜Q4 | 4問の合計 | 4〜20点 |
| **I型**（研究型） | Q5〜Q8 | 4問の合計 | 4〜20点 |
| **A型**（芸術型） | Q9〜Q12 | 4問の合計 | 4〜20点 |
| **S型**（社会型） | Q13〜Q16 | 4問の合計 | 4〜20点 |
| **E型**（企業型） | Q17〜Q20 | 4問の合計 | 4〜20点 |
| **C型**（慣習型） | Q21〜Q24 | 4問の合計 | 4〜20点 |

**点数が高いほど、そのタイプの志向が強い**ということ。

##### 6タイプそれぞれの意味

| タイプ | どんな人か | 向いている仕事 |
|--------|----------|--------------|
| **R型**（現実型） | 手を動かすのが好き。技術や機械に興味がある | エンジニア、製造、建設 |
| **I型**（研究型） | 分析や調べものが好き。データや理論に興味がある | 研究開発、コンサル、分析 |
| **A型**（芸術型） | 創造や表現が好き。自由な発想を大事にする | デザイナー、企画、クリエイティブ |
| **S型**（社会型） | 人の役に立つのが好き。対人支援に興味がある | 教育、福祉、人事 |
| **E型**（企業型） | リーダーシップを発揮したい。経営や組織運営に興味がある | 営業、マネジメント、起業 |
| **C型**（慣習型） | 正確さや管理が好き。堅実に物事を進める | 経理、法務、事務、品質管理 |

#### 特殊な質問: Q25〜Q51

| 設問 | 何を測るか | 回答形式 | どう使うか |
|------|----------|---------|----------|
| Q25〜Q36 | 行動特性（ポータブルスキル） | 各問1〜5点 | このうち**Q30（主体性）とQ31（変革意欲）**だけをフェーズ3で使う |
| Q37〜Q48 | 価値観（キャリア・アンカー） | 各問1〜5点 | 今回の判定では使わない |
| **Q49** | 嘘回答の傾向（L尺度） | 1〜5点 | 5点の場合、6タイプの点数を割り引く（フェーズ3のStep 0） |
| **Q50** | BtoB企業が好きか | 1〜5点 | 3点以上→BtoB企業リスト、3点未満→BtoC企業リストを出す |
| **Q51** | メーカー/インフラ企業が好きか | 1〜5点 | Q50と合わせて中西製作所様との相性を判定する |

### 回答結果の決まり方

6タイプの点数を高い順に並べて、**1位と2位**を取る。これが「この学生はどんなタイプか」の結論。

**具体例: ある学生の6タイプの点数**

| R型 | I型 | A型 | S型 | E型 | C型 |
|-----|-----|-----|-----|-----|-----|
| 8 | 6 | 7 | **18** | **10** | 5 |

→ 一番高い点数は**S型の18点** → これが**1位タイプ**
→ 二番目に高いのは**E型の10点** → これが**2位タイプ**
→ この学生は**「人の役に立ちたい（S型）」が一番強く、「リーダーシップ（E型）」も持っている**

**もう1つの例**:

| R型 | I型 | A型 | S型 | E型 | C型 |
|-----|-----|-----|-----|-----|-----|
| 10 | 8 | **16** | 12 | **19** | 7 |

→ 1位: **E型（19点）** → 2位: **A型（16点）**
→ この学生は**「経営・リーダーシップ（E型）」が一番で、「創造力（A型）」もある**

**ポイント**:
- **1位タイプ**がメインの推薦企業を決める基準になる
- **2位タイプ**が「あなたにはこういう面もありますよ」という補足の推薦企業になる
- 1位と2位が同点の場合は R → I → A → S → E → C の順で先にくる方が1位

### フェーズ1でできあがるもの（＝フェーズ2で送る11個の数値）

| 項目 | 値の例 | 何の点数か | フェーズ3のどこで使うか |
|------|--------|----------|---------------------|
| R | 8 | R型の合計点（4〜20点） | 1位・2位タイプの判定に使う |
| I | 6 | I型の合計点 | 同上 |
| A | 7 | A型の合計点 | 同上 |
| S | 18 | S型の合計点 | 同上 |
| E | 10 | E型の合計点 | 同上 |
| C | 5 | C型の合計点 | 同上 |
| Q30 | 4 | 主体性の点数（1〜5点） | 中西製作所様向きかの判定に使う |
| Q31 | 4 | 変革意欲の点数（1〜5点） | 同上 |
| Q49 | 3 | 嘘回答検知の点数（1〜5点） | 5点のときだけ6タイプの点数を補正する |
| Q50 | 4 | BtoB志向の点数（1〜5点） | BtoB企業 or BtoC企業の切り替えに使う |
| Q51 | 4 | メーカー志向の点数（1〜5点） | 中西製作所様との相性判定に使う |

> [!question] 会議で確認すべきポイント
> **Q: 6タイプの合計点はMIL様側で計算してから送るのか？**
> → 現状の想定: **MIL様側で合計済みの点数（例: R=15）を送る**。Dify側は合計された点数を受け取る設計で作っている。ただし、MIL様側で「生データ（Q1=5, Q2=4, …）を送りたい」という場合はDify側の受け口を変更する必要がある。
>
> **Q: 嘘回答補正（Q49が5のとき点数を×0.8する処理）はどちら側でやるか？**
> → 現状の想定: **Dify側（エヌイチが構築した処理の中）でやる**。MIL様は補正前の素の点数を送ればOK。

## フェーズ2: 検査結果をDifyに送る【MIL様 → Dify】

### このフェーズで何が起きるか

フェーズ1で算出された11個の数値を、MIL様のサーバーから中西製作所様のDifyに送る。送ると自動的にフェーズ3の判定処理が走り、結果が返ってくる（フェーズ4）。

**フェーズ2（送る）とフェーズ4（返ってくる）は1回のやり取り。** 電話でいうと、MIL様が電話をかけて質問し、Difyが電話口で答えを返す ── これが1回で完結する。

```
MIL様のサーバー                   中西製作所様のDifyアカウント
                                 （エヌイチが構築・運用）

  ① 11個の数値を送る ─────→    ② 受け取って判定処理を実行
                                  （フェーズ3の処理が動く）
  ④ 結果を受け取る   ←─────    ③ 判定結果を返す
                                  （フェーズ4）
```

### 送るために必要なもの

MIL様がDifyにデータを送るには、以下の**2つの情報**が必要。どちらも**中西製作所様のDifyアカウントから発行されるもの**で、エヌイチが代行してMIL様に共有する。

| 必要なもの | 何か（たとえ） | 誰が発行するか | 誰に渡すか |
|-----------|-------------|-------------|-----------|
| **接続先URL** | 「データの送り先住所」のようなもの | 中西製作所様のDifyアカウント | MIL様に共有 |
| **認証キー（APIキー）** | 「この人は使っていいですよ」という合鍵のようなもの | 中西製作所様のDifyアカウント | MIL様に共有 |

**準備の流れ**:
1. エヌイチが中西製作所様のDifyアカウント上でしくみを完成させる
2. Difyの管理画面で「公開」すると、**接続先URL**と**認証キー**が自動発行される
3. この2つをエヌイチからMIL様に共有する
4. MIL様はこの2つを自社サーバーに設定して、学生が検査を完了するたびにデータを送れるようにする

**MIL様が何かを発行してエヌイチや中西製作所様に渡す必要はありません。** 中西製作所様のDifyから出た情報を、MIL様が受け取って使うだけです。

### 送るデータの中身

フェーズ1でできた11個の数値をまとめて送る。技術的にはJSON（ジェイソン）というデータ形式で送信する。

```
送信データの中身（例）:

  R  = 8        ← R型の合計点
  I  = 6        ← I型の合計点
  A  = 7        ← A型の合計点
  S  = 18       ← S型の合計点
  E  = 10       ← E型の合計点
  C  = 5        ← C型の合計点
  Q30 = 4       ← 主体性の点数
  Q31 = 4       ← 変革意欲の点数
  Q49 = 3       ← 嘘回答検知の点数
  Q50 = 4       ← BtoB志向の点数
  Q51 = 4       ← メーカー志向の点数
```

全部で11個、すべて数値。

> [!question] 会議で確認すべきポイント
> **Q: データの項目名はこれで問題ないか？**
> → 現状の想定: R, I, A, S, E, C, Q30, Q31, Q49, Q50, Q51 という名前でDify側の受け口を作っている。MIL様側の実装で別の名前を使いたい場合は調整が必要。
>
> **Q: 認証キーの共有はいつ・どうやって行うか？**
> → 現状: まだ発行していない。Difyの有料プラン（Professional: 月額約9,000円）に切り替えた後に発行・共有する予定。共有方法（メール・Slackなど）は要相談。

## フェーズ3: 企業のマッチング【Dify / エヌイチ構築】

### このフェーズで何が起きるか

MIL様から受け取った11個の数値をもとに、**「この学生にどの企業を推薦するか」**を自動で判定する。

### 判定に使うもの

| 何を | 説明 |
|------|------|
| **11個の数値**（フェーズ2で受け取ったもの） | 学生の志向性タイプ・行動特性・業界志向 |
| **企業リスト**（Difyの中にあらかじめ登録してあるもの） | 7タイプ × BtoB/BtoC = 14パターンの企業名と推薦理由。津丸様提供のExcel「紹介企業」シートに基づき4/10に登録済み |

### 判定のしくみ（津丸様との確認に基づく確定ロジック）

判定は**4段階**あり、**上から順番にチェックしていく**。どこかのステップに当てはまったら、そこで結果が確定する。下のステップは見ない。

```
Step 0（前処理）: 嘘回答の補正
    ↓
Step 1（最優先）: 「志向が定まっていない」学生を検出
    → 当てはまったら → トモキャリに誘導して終了
    ↓ 当てはまらなかったら
Step 2: 「主体性・変革意欲が高い」学生を検出
    → 当てはまったら → 中西製作所様を推薦して終了
    ↓ 当てはまらなかったら
Step 3: 「中西と相性のいいタイプ＋業界志向」の学生を検出
    → 当てはまったら → 中西製作所様を推薦して終了
    ↓ 当てはまらなかったら
Step 4: タイプに合った企業を推薦
```

以下、各ステップを順番に説明します。

---

### Step 0: 嘘回答の補正（前処理）

**目的**: 自分をよく見せようとして回答を盛っている可能性がある学生の点数を割り引く。

**しくみ**:
- Q49（嘘回答検知）の点数が **5点（満点）のときだけ**、6タイプ（R, I, A, S, E, C）の点数すべてに**×0.8**をかける
- Q49が5点以外のときは、何もしない

**例**:

| 学生 | Q49の点数 | 補正前のE型 | 補正前のR型 | → 補正後のE型 | → 補正後のR型 |
|------|----------|-----------|-----------|-------------|-------------|
| Xさん | **5点** | 20点 | 15点 | **16点**（×0.8） | **12点**（×0.8） |
| Yさん | 3点 | 20点 | 15点 | 20点（そのまま） | 15点（そのまま） |

この補正後の点数を使って、以降のStep 1〜4を判定する。

> **現状**: Dify側（エヌイチが構築した処理の中）で自動的に補正する設計で作っている。

---

### Step 1: 志向が定まっていない学生を検出する（最優先）

**目的**: 6タイプの点数にほとんど差がない学生 ＝ どの仕事に向いているか判断できない学生を検出して、企業を推薦せずに**キャリア相談（トモキャリ）に誘導する**。

**条件**: 6タイプの中で一番高い点数と一番低い点数の差が **1点以下**

**例**:

| 学生 | R | I | A | S | E | C | 最高−最低 | 結果 |
|------|---|---|---|---|---|---|----------|------|
| Aさん | 12 | 12 | 11 | 12 | 12 | 11 | 12−11 = **1点** | → **トモキャリ誘導**（差がなさすぎて判定できない） |
| Bさん | 15 | 12 | 8 | 18 | 10 | 6 | 18−6 = **12点** | → 次のStep 2へ（はっきり差がある） |

→ Aさんのように全タイプの点数が横並びの場合は、企業を推薦しても意味がないのでトモキャリに案内する。

---

### Step 2: 主体性・変革意欲が高い学生を検出する

**目的**: 中西製作所様の社風に合う学生を検出して、**中西製作所様を推薦する**。

**背景（津丸様との確認に基づく）**: 中西製作所様は「安定した基盤がありながらベンチャー精神で変革を推進する」社風。この社風に合う人材は、**主体性（Q30）と変革意欲（Q31）が両方高い人**。

**条件**: Q30（主体性）が**4点以上**、かつQ31（変革意欲）が**4点以上**（両方とも5点満点中）

**例**:

| 学生 | Q30（主体性） | Q31（変革意欲） | 結果 |
|------|-------------|---------------|------|
| Aさん | **4** | **5** | 両方4以上 → **中西製作所様を推薦** |
| Bさん | **5** | 3 | Q31が3で4未満 → 次のStep 3へ |
| Cさん | 3 | **4** | Q30が3で4未満 → 次のStep 3へ |
| Dさん | 2 | 2 | 両方4未満 → 次のStep 3へ |

→ Q30とQ31の**両方が4以上**のときだけ中西製作所様と判定。片方だけ高くてもダメ。

---

### Step 3: 中西と相性のいいタイプ × 業界志向の学生を検出する

**目的**: Step 2に当てはまらなかったけど、**タイプ的にも業界志向的にも中西製作所様と合っている学生**を拾って、**中西製作所様を推薦する**。

2つの条件を**両方とも**満たす必要がある。

**条件①: タイプの相性**

1位タイプか2位タイプの**どちらかが E型・I型・A型 のいずれか**であること。

- **E型**（経営・リーダーシップ志向） → 中西製作所様が求める人材像と合う
- **I型**（研究・分析志向） → 中西製作所様が求める人材像と合う
- **A型**（創造・企画志向） → 中西製作所様が求める人材像と合う

1位でなくても、2位に入っていればOK。

**条件②: 業界志向の相性**

Q50（BtoB志向）とQ51（メーカー志向）の**平均が3点以上**であること。

- 平均3点以上 ＝ BtoB企業やメーカーに興味がある ＝ 中西製作所様の業界と合っている

**例**:

| 学生 | 1位 | 2位 | 条件① | Q50 | Q51 | 平均 | 条件② | 結果 |
|------|-----|-----|-------|-----|-----|------|-------|------|
| Aさん | **E型**(19) | A型(16) | ✅ 1位がE | 4 | 4 | 4.0 | ✅ | → **中西製作所様を推薦** |
| Bさん | S型(18) | **I型**(15) | ✅ 2位がI | 4 | 3 | 3.5 | ✅ | → **中西製作所様を推薦** |
| Cさん | S型(18) | R型(15) | ❌ S,Rは対象外 | 4 | 4 | 4.0 | ✅ | → 次のStep 4へ（条件①が×） |
| Dさん | **E型**(19) | A型(16) | ✅ 1位がE | 2 | 2 | 2.0 | ❌ | → 次のStep 4へ（条件②が×） |

→ Cさんはメーカー志向が高いが、タイプがS型・R型で中西製作所様の求める人材像（E/I/A）に該当しないので通過。Dさんはタイプは合うが、BtoB/メーカー志向が低いので通過。

---

### Step 4: タイプに合った企業を推薦する（通常判定）

**目的**: Step 1〜3のどれにも当てはまらなかった学生に対し、**その学生のタイプに合った企業を推薦する**。

**しくみ**: 以下の2つを出す。

1. **メインの推薦企業**: 1位タイプに対応する企業リストから出す
2. **補足の推薦企業**: 2位タイプに対応する企業リストから出す（「あなたにはこんな面もありますよ」という補足）

さらに、**Q50（BtoB志向）の点数**で、BtoB企業リストとBtoC企業リストを切り替える。
- Q50が **3点以上** → BtoB企業リストを使う
- Q50が **3点未満** → BtoC企業リストを使う

**具体例**: 1位 = S型、2位 = R型、Q50 = 4点 の学生の場合


## フェーズ4: 結果をMIL様に返す【Dify → MIL様】

### このフェーズで何が起きるか

フェーズ3の判定が終わると、その結果がMIL様に自動的に返される。

フェーズ2でMIL様がデータを送ったのと**同じ通信の「返事」として返ってくる**ので、MIL様側で特別な設定は不要。認証キーも不要（フェーズ2で送信したときに認証は済んでいるため）。

### 返ってくるデータの中身

判定結果として以下の8つの情報が返ってくる。MIL様はこれをもとに学生の結果画面を表示する。

| 項目 | 何が入るか | MIL様での使い方 |
|------|----------|---------------|
| **判定カテゴリ** | 「中西該当」「タイプ別マッチング」「トモキャリ誘導」のいずれか | どの結果画面を出すかの判別に使う |
| **1位タイプ** | R〜Cのいずれか | 「あなたは〇〇タイプです」として表示 |
| **2位タイプ** | R〜Cのいずれか | 「〇〇な面もあります」として補足表示 |
| **メインの推薦企業** | 企業名のリスト（最大5社） | 結果画面にメインとして企業名を表示 |
| **メインの推薦理由** | 各企業への推薦理由テキスト | 企業名の下に表示 |
| **補足の推薦企業** | 企業名のリスト（通常判定時のみ） | 「こんな企業も」として表示 |
| **補足の推薦理由** | 各企業への推薦理由テキスト | 同上 |
| **メッセージ** | 学生への一言メッセージ | 結果画面の冒頭 or 末尾に表示 |

### 判定カテゴリごとに返ってくる内容の違い

| | 中西該当（Step 2 or 3） | タイプ別マッチング（Step 4） | トモキャリ誘導（Step 1） |
