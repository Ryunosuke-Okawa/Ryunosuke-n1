# Difyセキュリティ構成まとめ（中西案件）

## 背景

中西製作所向けTOMOSHIKA（適性検査AIマッチング）のAPI連携において、Difyのセキュリティ構成について社内エンジニア（木村さん）からフィードバックを受けた内容を整理。

### 前提条件
- **個人情報は扱わない**（診断結果とDBの突合結果のみ）
- MILからJSONで適性検査結果を受信 → Difyで処理 → JSONで結果を返却
- 現在はDifyクラウド版（Sandbox）を使用中

---

## 2つの構成の比較

### 構成A: クラウド版（WebApp無効・API only）

現在の方針。すぐに使える。

```
[MILのサーバー]
      │
      │ APIキーで認証
      ▼
[Dify クラウド（dify.ai）]
  └─ WebAppは無効化（API onlyで運用）
      │
      ▼
[MILのサーバー → 学生に表示]
```

| 項目 | 内容 |
|------|------|
| Dify料金 | Professional $59/月（約9,000円） |
| サーバー | 不要（Dify社が管理） |
| セキュリティ | APIキー認証のみ。IP制限は不可 |
| 運用保守 | Dify社が対応 |
| 導入の手軽さ | すぐに使える |

**セキュリティの考え方**
- WebAppを無効にすることで、ブラウザからの直接アクセスは不可になる
- ただしAPIのエンドポイント（接続先URL）自体はインターネット上に存在する
- APIキーが漏洩した場合、誰でもアクセスできてしまうリスクがある
- 個人情報を扱わないため、万が一漏洩しても深刻な被害にはならない

---

### 構成B: セルフホスト版（IP制限あり）

より高いセキュリティが必要な場合の選択肢。

```
[MILのサーバー（IPアドレス固定）]
      │
      │ IP制限 + APIキーで認証
      ▼
[自社サーバー（VPS）]
  ├─ Dify本体（API + ワークフロー）
  ├─ PostgreSQL（データベース）
  ├─ Redis（キャッシュ）
  └─ Nginx（通信の入口・IP制限をここで設定）
      │
      ▼
[MILのサーバー → 学生に表示]
```

| 項目 | 内容 |
|------|------|
| Dify料金 | 無料（Community Edition） |
| サーバー | VPSの契約が必要（月額2,000〜12,000円） |
| セキュリティ | APIキー認証 ＋ IP制限が可能 |
| 運用保守 | 自社で対応（木村さん等） |
| 導入の手軽さ | サーバー構築・設定が必要 |

**セキュリティの考え方**
- 自社サーバーにDifyを設置するため、アクセスできる相手をIPアドレスで制限できる
- MILのサーバーのIPだけを許可すれば、それ以外からは一切アクセスできない
- APIキーが漏洩しても、許可されたIP以外からはアクセス不可

---

### セルフホスト版に必要な契約・準備

| # | 必要なもの | 何のため | 月額目安 | 契約先の例 |
|---|---|---|---|---|
| 1 | VPSサーバー | Difyを動かすサーバー | 2,000〜12,000円 | ConoHa VPS / さくらのVPS / AWS |
| 2 | ドメイン | APIの接続先URL | 100〜200円 | お名前.com 等 |
| 3 | SSL証明書 | 通信の暗号化（https化） | 無料 | Let's Encrypt |
| 4 | LLM API（任意） | AI判定を使う場合のみ | 使用量次第 | OpenAI等 |

> VPSとは、インターネット上にサーバーを1台借りて、自由にソフトを入れて使えるサービス（Xサーバーの自由度が高い版のようなもの）

## 現時点の方針（木村さん・髙橋さん判断）

**構成Aのクラウド版（WebApp無効・API only）で進める**

- 個人情報を扱わないため、過度なセキュリティ投資は不要
- 先方のセキュリティポリシー次第で、構成Bへの移行を検討

---

## 中西側に確認すべき事項

### 1. セキュリティポリシー
- 外部API連携に関するセキュリティ要件はあるか（IP制限必須、セキュリティチェックシート提出等）
- 社内の情報セキュリティ部門の審査プロセスがあるか（ある場合、審査期間）

### 2. アクセス元の構成
- APIを利用するのはMILのサーバーか、中西製作所のサーバーか
- IP制限をかける場合、固定IPでの運用は可能か

### 3. JSON連携仕様（保留事項の回収）
- MIL → エヌイチに送るJSONの項目・形式の確定時期
- L尺度補正・偏差値化の担当範囲（MIL側 or エヌイチ側）

### 4. 将来の個人情報取り扱い
- 今後、学生の個人情報（氏名・メール等）をAPI経由で扱う可能性はあるか
- ある場合、最初からセルフホスト版で構築した方が二度手間を避けられる

### 5. 本番稼働スケジュール
- 9月末本番稼働の目標に変更はないか
- セキュリティ審査等のゲートがあるか（4月末仮納品に影響するか）
