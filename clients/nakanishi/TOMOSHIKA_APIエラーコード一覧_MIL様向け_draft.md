## 1. エラーレスポンスの基本構造

エラー発生時は HTTPステータスコード（2xx以外）とともに、以下のJSONが返されます。

```json
{
  "status": 400,
  "code":    "invalid_param",
  "message": "R must be between 4 and 20"
}
```

| キー | 説明 |
|---|---|
| `status`  | HTTPステータスコード |
| `code`    | エラー種別を示す識別子 |
| `message` | エラーの詳細メッセージ（英語） |

## 2. エラーコード一覧

### 2.1 認証・接続系

| HTTP | code | 意味 | 主な原因 | MIL様側の対処 |
|------|------|------|---------|--------------|
| `401` | `unauthorized` | 認証失敗 | APIキーが無効・未設定・期限切れ | APIキーの設定確認。問題あればエヌイチに再発行依頼 |
| `403` | `forbidden`    | アクセス権限なし | APIキーが本ワークフローの実行権限を持たない | エヌイチに連絡。権限再設定を依頼 |
| `404` | `not_found`    | エンドポイント不在 | URLの誤り、ワークフロー削除 | URLの再確認。エヌイチに連絡 |

### 2.2 リクエスト不正系

| HTTP | code | 意味 | 主な原因 | MIL様側の対処 |
|------|------|------|---------|--------------|
| `400` | `invalid_param`          | 入力パラメータ不正 | 必須項目の欠損、型違反、範囲外値（R〜Cが4〜20外、Q30〜Q51が1〜5外を含む） | 入力データをバリデーション後に再送。点数算出ロジックの確認 |
| `400` | `app_unavailable`        | アプリが利用不可 | ワークフローが公開停止状態 | エヌイチに連絡 |
| `400` | `workflow_request_error` | ワークフロー実行失敗 | Dify内部処理エラー | リクエスト内容をログに残したうえでエヌイチに連絡 |

### 2.3 レート制限・リソース系

| HTTP | code | 意味 | 主な原因 | MIL様側の対処 |
|------|------|------|---------|--------------|
| `429` | `too_many_requests`       | レート制限 | 短時間の大量リクエスト | 指数バックオフでリトライ（初回1秒・最大30秒） |
| `429` | `quota_exceeded`          | 月次上限超過 | 月間5,000件の上限到達 | エヌイチ＋中西製作所様に連絡。プラン変更の要否を協議 |
| `400` | `provider_quota_exceeded` | LLMクレジット不足 | Dify内で使うLLMの残高切れ（※本案件では原則発生しない） | エヌイチに連絡 |

### 2.4 サーバー系

| HTTP | code | 意味 | 主な原因 | MIL様側の対処 |
|------|------|------|---------|--------------|
| `500` | `internal_server_error` | Dify内部エラー | Dify基盤側の一時障害 | 30秒後にリトライ。継続する場合はエヌイチに連絡 |
| `502` | `bad_gateway`           | ゲートウェイエラー | ネットワーク一時障害 | 同上 |
| `503` | `service_unavailable`   | サービス利用不可 | Difyメンテナンス等 | 同上 |
| `504` | `gateway_timeout`       | タイムアウト | 処理時間超過 | タイムアウト値を60秒に延長のうえリトライ |

## 3. 推奨されるMIL様側の実装

### 3.1 リトライポリシー

| HTTPステータス | リトライすべきか | 戦略 |
|---|---|---|
| `4xx`（400, 401, 403, 404） | ❌ する意味なし | エラーログを残して即時エラー扱い |
| `429` | ✅ する | 指数バックオフ（1s → 2s → 4s → 8s → 最大30s） |
| `5xx`（500, 502, 503, 504） | ✅ する | 30秒間隔で最大3回 |

### 3.2 学生への画面表示

| 発生エラー | 学生に見せる文言（例） |
|---|---|
| `4xx` 全般 | 「一時的にサービスが利用できません。しばらくしてから再度お試しください。」 |
| `429` / `5xx` | 「混雑しております。少し時間をおいてから再度お試しください。」 |
| タイムアウト | 同上 |

※ エラーコード・詳細メッセージは**学生には見せない**運用を推奨します（原因はMIL様側のログに残す）。

### 3.3 重大事象発生時の連絡先

以下のいずれかに該当した場合、**エヌイチおよび中西製作所様の両方**にご連絡ください。

- `401` / `403` が継続する
- `quota_exceeded` が発生した
- `5xx` が10分以上継続する
- `workflow_request_error` が特定の入力パターンで再現する
