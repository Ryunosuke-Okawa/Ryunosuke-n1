# TOMOSHIKA API 出力結果の見方

適性検査（51問）の回答をもとに、システムが以下の8項目を返します。
テスト実行時の出力結果を例に、各項目の意味をご説明します。

---

## 出力項目と意味

- **result_type**（判定カテゴリ）→ 今回: `nakanishi`
  - この学生がどのカテゴリに分類されたかを示す項目
  - `nakanishi` = 中西製作所に適性あり / `type_match` = 他の企業を推薦 / `tomocari` = 志向未分化→トモキャリ誘導
  - → 今回は**中西製作所に適性あり**と判定

- **primary_type**（第1位の志向性タイプ）→ 今回: `S`（社会型）
  - 6タイプ（R:技術, I:研究, A:創造, S:対人支援, E:リーダーシップ, C:管理）のうち最もスコアが高かったもの
  - → この学生は**対人支援・教育志向**が最も強い

- **secondary_type**（第2位の志向性タイプ）→ 今回: `R`（現実型）
  - 2番目にスコアが高かったタイプ。「こんな側面もあります」という補足情報
  - → 社会型が主軸でありつつ、**ものづくり・技術志向**も持ち合わせている

- **primary_companies**（推薦企業リスト）→ 今回: 中西製作所 / キーエンス / 平田機工 / マクニカ / ダイフク
  - この学生に推薦する企業の一覧。中西該当の場合は中西製作所を筆頭に同系統企業を表示
  - → 今回はBtoB志向が高いため**BtoB企業リスト**が選ばれている（BtoC志向が高い場合はソニー・リクルート等に切り替わる）

- **primary_reasons**（推薦理由）→ 今回: 5社分の理由テキスト
  - 各企業について「なぜあなたに合っているか」を説明する文章
  - → 結果画面で企業名の下に表示する想定

- **secondary_companies / secondary_reasons**（補足企業・理由）→ 今回: どちらも「該当なし」
  - 通常判定の場合は2位タイプに対応する企業が入るが、中西該当の場合は**中西グループへの誘導を優先するため表示しない**

- **message**（学生へのメッセージ）→ 今回: 「あなたの主体性と変革意欲は、安定基盤×ベンチャー志向の企業で大きく花開きます。」
  - 判定結果に応じた締めの一言。結果画面の冒頭 or 末尾に表示する想定
