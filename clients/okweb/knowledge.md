## 先方の課題（商談で確認済み）

### 最優先課題: 求人マッチング業務の工数
- 求職者との面談後、求人情報とのマッチング・ピックアップ作業が**1件あたり1〜1.5時間**
- 1日の対応可能件数は最大4〜5件が限界で、事業拡大の障壁
- 使用媒体: **CIRCUS（サーカス）**、**もう1社（スカウトサービス、名称要確認）**
- 職種・業界知識が必要で精神的負荷も高い
- 属人化しており、標準化ができていない

### 二次課題
- 求職者1人ひとりに合わせたスカウトメール文面の個別作成・送付
- 集客数拡大・返答率向上のために取り組みたい

### 将来課題
- Notionに面談履歴を蓄積・記録し、次回紹介時に活かすワークフロー構築

### 先方の希望
- 第1優先: 求人マッチング〜メール下書き生成の自動化
- 第2優先: スカウトメールのパーソナライズ送付
- 方向性: 最初はエヌイチに構築してもらいつつ、**自分たちで修正・追加できる内製化**を目指す
- システム環境: Microsoftベース（Teams / Outlook / OneDrive）

## 人材紹介でのAI活用方法

### 想定エージェントフロー
```
【トリガー】CAが「求職者名」を入力
  |
STEP1 | Notion API → 求職者情報自動取得
  |     （履歴書・職務経歴書・面談文字起こし）
  |
STEP2 | Claude Sonnet → 求職者プロファイル生成
  |     （スキル・経験・希望条件・強み・懸念点を構造化）
  |     （CAの暗黙知をプロンプトに組み込み済み）
  |
STEP3 | Dify RAG → 求人候補の一次抽出
  |     （意味的に近い求人を上位20件自動抽出）
  |
STEP4 | Claude Sonnet → スコアリング・絞り込み
  |     （20件→3〜5件に絞り込み）
  |     （各求人: 推薦理由・メリット・デメリット・懸念点）
  |
STEP5 | Notion自動格納 + Slack通知
```

### 技術的なポイント
- **HERP Hire APIは使えない**（採用企業側の権限であり、人材紹介会社には非対応）
- **ジョブミル連携プランCSV + RAG + n8n** が現実的な自動化ルート
- APIが取得できれば完全自動化可能、CSVの場合は更新部分だけ手動（ほぼ自動に近づけられる）
- Dify単体で90%カバー可能。完全自動化にはn8nの追加が必要

### ツール構成
| ツール | 役割 |
|--------|------|
| Dify | RAGナレッジ（求人DB）・AI処理・チャットUI |
| n8n | 定期実行・外部API連携・CSV取得・通知 |
| Notion API | 求職者情報取得・結果格納 |
| Claude API（Sonnet） | 求職者分析・マッチングスコアリング |
