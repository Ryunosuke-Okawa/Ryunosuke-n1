# 半期振り返り — AI活用実績（2025年10月〜2026年3月）

## サマリー

AI活用は「壁打ち・ディスカッション」だけにとどまらず、以下7カテゴリで**具体的な制作物・システム・成果**を生み出した。特に2026年3月にClaude Code（COOシステム）を構築して以降、AI活用の密度と具体性が大幅に向上している。

| カテゴリ | 主な成果物数 | 事業インパクト |
|---------|------------|-------------|
| ① 制作物（提案書・LP・PR記事） | 5件 | 営業ツール整備 → 提案品質向上 |
| ② システム開発・自動化 | 11スクリプト + Difyワークフロー | 業務工数削減 + クライアント納品物 |
| ③ 議事録・ナレッジ管理 | 3件（自動生成） | 情報共有の即時化 |
| ④ 営業支援（パイプライン管理・メール） | 20案件+ | 営業活動の効率化 |
| ⑤ 法務・財務 | 契約書レビュー + PL管理 | リスク低減 + 経営判断基盤 |
| ⑥ 戦略立案・壁打ち | SNS戦略・ローンチ計画 | 意思決定の迅速化 |
| ⑦ 業務基盤整備 | Claude Code COOシステム | AI活用の組織インフラ化 |

## ② システム開発・自動化

### A. TOMOSHIKA 採用AIマッチング（中西製作所案件）

**概要**: 51問の適性検査 → RIASEC 6タイプ分類 → 4段階判定 → 企業マッチング の全自動フロー

| 項目 | 内容 |
|------|------|
| 技術スタック | Dify（ワークフロー） + Python（判定ロジック） |
| ロジック | L尺度補正 → 未分化判定 → 行動特性 → タイプ×志向 の優先順位判定 |
| 企業マスタ | 7タイプ × BtoB/BtoC × 大手/中小 = 全70社のマッチングDB |
| テスト | 複数パターンで動作確認済み（4/10） |
| コスト設計 | 3プラン比較シミュレーションを作成・津丸さんに共有済み |

**AIの役割**: ロジック設計（テーブル構造・判定フロー・分岐条件の最適化）、Pythonコード生成、コストシミュレーション計算

### B. 新規事業PL スプレッドシート自動化

Google Sheets API × Pythonで、売上・原価・粗利の自動集計システムを構築。

| スクリプト | 用途 |
|-----------|------|
| `pl_reform.py` | 仕訳帳型→マトリクス形式変換 |
| `rebuild_sales_pl.py` | v2: 2シート構成（月次サマリー＋売上明細）に再構築 |
| `rebuild_case_pl.py` | 案件別P/L半自動化 |
| `add_cost_summary.py` | コスト一覧シート追加 |
| `rebuild_cost_sheet.py` | コスト管理シート再構築 |
| `restore_cost_simple.py` | シンプルマトリクス復元 |
| `fix_colors_dropdowns.py` | プルダウン削除＋文字色修正 |
| `fix_all_colors.py` | 全シート書式完全修正 |

**AIの役割**: SUMIFS・INDEX/MATCHの複雑な集計フォーミュラ自動生成、Google Sheets APIの直接操作コード生成、v1→v2の設計判断支援

### C. その他の自動化ツール

| ツール | 用途 | 技術 |

## ⑤ 法務・財務

### 契約書レビュー支援

中西製作所向けAI活用業務委託契約書の作成において、AIで初稿を作成し、蓮見弁護士のレビューを経て以下を学習・反映:

1. **完了条件**: 「約7割」→ 4つの具体的フロー動作基準に変更

3. **ライセンス失効条項**: 未払い解除時の対応を明記

→ これらの教訓を `knowledge/contract_legal_lessons.md` にナレッジとして蓄積。今後の契約書作成に適用可能。

## ⑦ 業務基盤整備（Claude Code COOシステム）

2026年3月2日にClaude Codeを「AI COO」として構築。以下のインフラを整備:

### 構築したもの

| 項目 | 内容 |
|------|------|
| **CLAUDE.md（指示書）** | 役割定義・ワークフロー・ディレクトリ構造・外部ツール連携の一元管理 |
| **Obsidian vault構造** | strategy / finance / clients / marketing / knowledge / tools の体系的ディレクトリ |
| **外部ツール連携（MCP）** | Google Drive / Docs / Sheets / Slides / Calendar / Notion の認証・連携完了 |
| **自動スキル** | `/daily-schedule`（朝の工程表）、`/mail-check`（未返信チェック）、`/inbox-organize`（受信トレイ整理）、`/meeting-minutes`（議事録生成）、`/write-article`（執筆チーム）等 |
| **定期実行** | 毎朝8時の工程表自動生成（Cron） |

### 効果

- 大川がコンテキストを毎回説明しなくても、AIがvault内の情報を自動で参照して回答
- 成果物が指示なしでもObsidianに自動保存される
- 議事録・提案書・メール下書き等のワークフローが**スキル化**され、再現可能に

## 「壁打ち」と「制作物」の比率（体感）

振り返ると、AI活用は**壁打ちだけではなかった**。

- **壁打ち・ディスカッション**: 戦略方針の議論、案件対応の相談、技術的な検討
- **制作物**: 提案書5件、自動化スクリプト11本、議事録3件、LP文案、PRTimes記事、コストシミュレーション、契約書初稿、ガントチャート
- **システム構築**: Claude Code COOシステム、TOMOSHIKA Difyワークフロー、PL自動集計

**特に3月のClaude Code導入以降**、壁打ちの結果がそのまま制作物として保存・蓄積されるようになり、「相談して終わり」から「相談→制作→保存→再利用」のサイクルが回り始めた。
