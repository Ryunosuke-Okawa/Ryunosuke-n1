## 7. 既にあるたたき（百奏音さんに渡す素材）

4/14タスク整理Step2より、大川側で既に骨子を作成済み。百奏音さんはこれをベースに構成を切り直す想定。

### 流入経路タグ設計案（**購入者までトレースするための全ステップ設計**）

**設計思想**：LINE友だち追加時に付与したタグを、**有料課金到達まで引き継ぐ**。各ステップで識別子を失わないよう、4層構造で設計。

| ステップ | 識別手段 | 付与タイミング |
|---|---|---|
| ①SNS投稿 | UTMパラメータ（Source/Medium/Campaign） | 投稿時にリンク埋込 |
| ②LINE登録 | LINE流入タグ（UTMと一致させる） | 友だち追加時 |
| ③LP遷移 | UTM引き継ぎ（LINEからLPへのリンク） | LINE配信時 |
| ④Kawaru登録 | メール＋電話番号でLINEユーザーと突合 | 登録フォーム |
| ⑤有料課金 | Kawaru DB上の契約情報 × LINEタグ | 決済完了時 |

**流入元タグ（UTM＋LINEタグで統一）**：

| 流入元 | タグ名案 |
|---|---|
| YouTube | `utm_youtube` |
| Instagram | `utm_instagram` |
| X | `utm_x` |
| TikTok | `utm_tiktok` |
| 取りこぼしセミナー | `utm_seminar` |
| LP直接 | `utm_lp` |
| PRタイムズ | `utm_pr` |
| ハウスリスト | `utm_houselist` |
| アフィリエイト／代理店 | `utm_affiliate`（6/1以降想定） |

### 登録時アンケート項目

- メールアドレス（必須）← Kawaru側との突合キー
- 電話番号（必須）
- 企業名（任意）
- 役職（任意）

1. **生成AI基礎研修ガイド**（既存資産・一次特典として投入済み実績あり）
