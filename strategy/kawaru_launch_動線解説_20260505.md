# Kawaru β版ローンチ 動線解説書

## 0. このドキュメントの目的

Kawaru β版ローンチ（2026/5/26）に向けて構築した**集客→登録→有料化までの動線**について、「**なぜこの設計になったのか**」と「**どう動くのか**」を関係者全員が同じ理解で把握できるように整理した解説書。

詳細仕様は別ドキュメント（[[kawaru_launch_doosen_整理_20260504]] / [[kawaru_launch_tags_banners_20260504.md]]）を参照。

---

## 1. 動線の全体像

```
[フロント]
  IG／YT／X／一斉配信
        ↓
  ハブ経由 or 直B
        ↓
┌─────────────────────────────┐
│         B LINE              │
│  ↓ リッチメニュー（4択）     │
│  ① 無料トライアル           │
│  ② 個人/法人LP（タグ振分）   │
│  ③ セミナー予約             │
│  ④ Kawaru情報              │
└─────────────────────────────┘
        ↓
  個人向け／法人向けシナリオ（5通）
        ↓
  セミナー予約 → セミナーシナリオ → セミナー実施
        ↓
  Kawaruアカウント発行 → クレカ登録 → アカウント登録（完了）
        ↓
  2週間トライアル → 有料化

  ※ 別ルート：法人向けLP / 法人シナリオ5通目 → TEAMプラン問い合わせ → 商談
```

---

## 2. 動線設計の3つの軸

### 軸① フロントは媒体・コンテンツ別に「ハブ経由」と「直B」を使い分ける

| 動線 | 使う場面 |
|---|---|
| **ハブ経由** | 通常コンテンツ（スクール側資産を維持しながらKawaruに導く） |
| **直B** | Kawaru訴求を主目的とするコンテンツ（最短でB LINEへ） |

**理由**：
- 通常コンテンツまで全部Bに直流入させると、スクール側のSNS資産（顧客リスト・既存導線）を毀損する
- 一方でKawaru訴求コンテンツをハブ経由にすると、間で離脱するし「Kawaruに興味ある人」を識別できない

→ **コンテンツの目的に合わせて使い分け**ることで、両方の効率を最大化する

### 軸② B LINE到達後はシナリオを「個人向け」「法人向け」で2分岐させる

**理由**：
- 個人向け（toC）と法人向け（toB）で訴求が真逆
  - 個人：楽／稼ぐ／時短／自分のタスク削減
  - 法人：業務効率化／組織課題解決／DX
- 同じシナリオを送ると、両方に刺さらない中途半端なメッセージになる
- 流入元（チャンネル特性）からどちらに入れるべきかは自動判定できる

→ **2本立てに分けて、それぞれを最大化**する

### 軸③ ユーザーに2択を提示する（押し付けない）

B LINEリッチメニューでは「**すぐ使う**」と「**詳しく見る（LP）**」を併設。

**理由**：
- ユーザーの温度感は流入時点では分からない
- LPを必ず挟むと「もう登録したい」層が離脱する
- LPなしで登録のみだと「もっと知りたい」層が不安で離脱する
- → 二択を出して**ユーザー自身に選ばせる**のが最も離脱を抑える

## 3. フロント（媒体別動線）

### Instagram

| コンテンツ | 動線 | 意味 |
|---|---|---|
| 通常リール | IGハブ経由 | スクール既存導線を維持しつつ、リッチメニューで誘導 |
| Kawaruプロモーションリール | 直B | Kawaru訴求が主目的なので最短経路 |
| ストーリー | 直B | 24時間限定の即時訴求 |
| ハイライト | 直B | プロフィールに常設の動線（ストーリー残し） |
| istep | 直B | DM自動応答からの誘導 |

### YouTube（チャンネル別に役割分担）

| チャンネル | 性格 | 動線 | シナリオ |
|---|---|---|---|
| **AI収益化ラボ** | 個人向け（稼ぐ・副業）。河村運営 | 通常長尺はハブ経由／Kawaru訴求は直B | 個人向け |
| **ClaudeCodeチャンネル** | 業務効率化・開発者向け。大川共同運営 | 全部直B | **法人向け** |
| **ゆうチャンネル** | ClaudeCodeチャンネルと同性格 | 全部直B | **法人向け** |

**ポイント**：チャンネル特性そのものが「個人/法人」を決める。**サムネ・タイトル戦略はチャンネルの色で変える**。

### X

| 担当 | コンテンツ | 動線 | シナリオ |
|---|---|---|---|
| 田中 | 法人向け記事 | 直B | 法人向け |
| 奥山 | 10投稿シリーズ（経営者・DX担当者向け） | 直B | 法人向け |

→ **Xは全部B直行・法人寄せに振り切る**

### 一斉配信（5/24-25）

| 配信元 | 内容 | 合流先 |
|---|---|---|
| 受講生LINE | シナリオ4通 | B LINE（個人向けタグ） |
| ハウスリスト | シナリオ4通 | B LINE（個人向けタグ） |

→ 既存LINE上で「Kawaruについて4通配信→興味ある人がB LINEに移行」する温め動線。
→ どちらも個人向けタグで合流。

## 4. B LINE到達後の流れ

### リッチメニュー（4択）

B LINE登録直後に表示される4つのボタン：

| ボタン | 飛び先 | 想定ユーザー |
|---|---|---|
| ① **無料トライアル** | Kawaruアカウント発行ページに直行 | すぐ使いたい層 |
| ② **個人向けLP** | 個人向けLP（個人タグの人にだけ表示） | 詳しく見たい個人ユーザー |
| ③ **法人向けLP** | 法人向けLP（法人タグの人にだけ表示） | 詳しく見たい法人ユーザー |
| ④ **セミナー予約** | セミナー予約フォーム | じっくり聞きたい層 |

**※ ②と③はタグで自動振り分け**。個人タグの人には個人LPボタンだけ、法人タグの人には法人LPボタンだけが表示される。

### シナリオ配信（5通ずつ）

- **個人向けシナリオ**：5通（楽／稼ぐ／時短軸）
- **法人向けシナリオ**：5通（業務効率化／DX軸）
- **毎通B LINEのCTAを併設**（登録・LP・セミナー予約への導線）
- 各シナリオの**最終地点はセミナー予約**
- **法人向けシナリオ5通目**からは別途「TEAMプラン問い合わせ」への導線も併設

### セミナーシナリオ（個人・法人統合）

- 個人/法人どちらからもセミナー予約に合流
- セミナー予約後にセミナーシナリオ5通配信（リマインド・期待醸成）
- セミナー実施後にKawaruアカウント発行への誘導

### Kawaruアカウント発行〜有料化

```
アカウント発行（仮） → クレカ登録 → アカウント登録（本登録完了） → 2週間トライアル → 有料化
```

**注意**：「アカウント発行」は仮発行。**アカウント登録**まで完了して初めて使える状態になる仕様。

## 5. 法人ルート（個人と分岐する動線）

法人ユーザーは**有料化まで2ルート**ある：

```
法人向けシナリオ
   ├→ セミナー予約 → セミナー → アカウント発行 → 有料化（個人プラン相当）
   └→ TEAMプラン問い合わせ → 商談（団体契約）
                ↑
        法人向けLP内CTAから直接来る場合もあり
```

**意味**：法人ユーザーには「個人で試して気に入ったら自社導入を相談する」と「最初から組織導入を相談する」の両方の入口がある。

## 6. タグ設計の意味（最小限の説明）

タグの詳細仕様はLステップ構築者にお任せ。以下は設計思想だけ。

### タグの2層構造

| 種類 | 役割 | 例 |
|---|---|---|
| **流入元タグ** | どこから来たか（記録） | `IG_リール_通常リール` |
| **セグメントタグ** | 今どんな状態か（管理） | `個人向け`／`セミナー予約済み` |

### 自動付与の流れ

```
ユーザーがリンクから流入
   ↓
URLパラメータで流入元タグが自動付与
   ↓
流入元タグから「個人向け／法人向け」セグメントタグも自動付与
   ↓
セグメントタグを見てシナリオが自動発火
```

**例**：IGの通常リールから来た人 → `IG_リール_通常リール` タグ ＋ `個人向け` タグが自動で付与 → 個人向けシナリオ1通目が即配信

### 流入経路ごとに識別子を増やす

各リール／各動画で個別に識別したい場合は、**ベースタグに日付＋テーマを追加**する。

例：`IG_リール_通常リール_260520_AI効率化術`

→ どのリールから来たかを後で分析できる。
