# Kawaru β版ローンチ 流入タグ・バナー・リッチメニュー設計

## 1. タグ設計

### 1-1. タグの2層構造

| 種類 | 役割 | 例 |
|---|---|---|
| **流入元タグ** | 「どこから来たか」の記録（履歴） | `IG_リール_通常リール` |
| **セグメントタグ** | 「今どんな状態か」の管理（ステータス） | `個人向け` `セミナー予約済み` |

**設計思想**：
- 流入元タグ → そこから自動でセグメントタグを付ける（個人向け／法人向け振り分け）
- 状態が変わるたびにセグメントタグを積み重ねていく
- シナリオはセグメントタグを参照して発火する

### 1-3. セグメントタグ（状態管理）

| タグ名 | 付与タイミング | 用途 |
|---|---|---|
| `個人向け` | 流入時自動付与 | 個人向けシナリオ発火 |
| `法人向け` | 流入時自動付与 | 法人向けシナリオ発火 |
| `LP訪問済み_個人` | 個人向けLP訪問時 | LP見て登録しなかった人へのフォロー |
| `LP訪問済み_法人` | 法人向けLP訪問時 | 同上 |
| `セミナー予約済み` | セミナー予約完了時 | セミナーリマインド配信 |
| `セミナー参加済み` | セミナー出席時 | 参加後フォロー配信 |
| `Kawaruアカウント登録済み` | アカウント発行時（メール／電話突合） | シナリオ停止／CS系配信に切替 |
| `トライアル中` | 2週間トライアル開始時 | トライアル活用Tips配信 |
| `有料転換済み` | 決済完了時 | カスタマーサクセス配信 |
| `解約` | 解約時 | リターゲティング配信 |

---

### 1-4. 大進への依頼項目（タグ周り）

1. **全流入元タグをLステップに登録**（上記一覧）
2. **全セグメントタグをLステップに登録**
3. **「流入元タグ付与 → セグメントタグ自動付与」のルール構築**
4. **シナリオ配信条件を「セグメントタグ」で記述**
   - `個人向け` → 個人向けシナリオ1通目発火
   - `法人向け` → 法人向けシナリオ1通目発火
5. **状態変化イベントでタグ自動更新**
   - LP訪問／セミナー予約／アカウント登録／解約
6. **タグ別レポート**（流入元別CV率・有料転換率の可視化）
