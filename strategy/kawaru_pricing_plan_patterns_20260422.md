# Kawaru料金プラン イメージ4パターン

**前提**：大川PMの方針（ライセンス付与／しない軸、2プラン維持、テンプレ共有＝組織プラン限定、5名下限はこだわりなし、価格感は仮置き、β版改定前提OK）を踏まえて設計。

---

## 🎯 4パターンの1行サマリー

| パターン | 一言で言うと | 大川PM方針との親和性 |
|---------|------------|------------------|
| **A. 現行案ベース（微修正）** | 今の設計を維持し、表現だけ整える | 100%（完全維持） |
| **B. Yoom Business型（定額）** | チームを定額化、人数上限 or 無制限に | 高（業界トレンドと整合） |
| **C. パッケージ＋追加席型** | 「5席込み＋追加席課金」で加算的に見せる | 高（価格レンジ維持できる） |
| **D. ワークスペース軸型（Jinba型）** | 人以外の軸で組織らしさを表現 | 中（新概念の導入が必要） |

## パターンA：現行案ベース（微修正）

### プラン

```
┌─────────────────────────────────────────────┐
│ TRIAL          │ PERSONAL       │ TEAM       │
│ 無料           │ ¥9,000 /月     │ ¥14,900    │
│                │                │ /人/月     │
│ 14日間         │ 1名            │ 5名〜      │
│ 1名            │ 無制限ワークフロー│ 無制限     │
│ テンプレ利用可 │ テンプレ利用可 │ テンプレ共有│
└─────────────────────────────────────────────┘
```

### 狙い
今の設計を維持。表現の微調整のみ。「5名〜」を「組織プラン（最低5名から）」のようにフレーミングだけ改善。

### メリット
- 意思決定が最も早い（既存設計の踏襲）
- 開発・CS・LP改修がほぼ不要
- 高単価で利益率確保しやすい

### デメリット
- 業界トレンドと逆行（座席課金×高単価×下限表記）
- PERSONAL→TEAMが8.3倍ジャンプで離脱リスク大
- 3〜4名チームを取りこぼす
- β版で顧客から「5名の必然性」を必ず問われる

### Yoom比較
| | Kawaru TEAM | Yoom Business |
|---|---|---|
| 5名月額 | ¥74,500 | ¥15,000（$100） |
| 価格差 | Yoom比**約5倍** | — |

## パターンB：Yoom Business型（定額・人数上限）

### プラン

```
┌──────────────────────────────────────────────────┐
│ FREE          │ PERSONAL      │ TEAM            │
│ 無料          │ ¥9,000 /月    │ ¥49,000 /月     │
│               │               │                 │
│ 月100実行     │ 1名           │ 10名まで        │
│ 1名           │ 無制限実行    │ 無制限実行      │
│ ワークフロー3個│ 組織共有なし  │ 組織テンプレ共有│
└──────────────────────────────────────────────────┘
         ※ 10名超は ENTERPRISE（要相談）
```

### 狙い
「ライセンス付与する／しない」方針をYoom思想で素直に実装。TEAMは定額化し、1人あたり計算と下限表記を両方撤廃する。

### メリット
- **業界トレンドと整合**（Yoom Business、Zapier Team と同設計思想）
- 1人あたり計算が無いので「なぜ5名？」問題が消える
- 稟議で「ツール代¥49,000／月」と単独行で処理できる
- 社内展開の摩擦ゼロ（8名チームも10名チームも同額）
- 無期限無料で「試す壁」を業界標準レベルに下げる

### デメリット
- 10名までなら客単価が下がる（8名ならパターンAの56%）
- 大規模顧客（20名以上）はENTERPRISEに振る必要
- 利益率モデルの再計算が必要

### Yoom比較
| | Kawaru TEAM | Yoom Business |
|---|---|---|
| 月額 | ¥49,000 | ¥15,000（$100） |
| 人数 | 10名まで | 無制限 |
| 価格差 | Yoom比**3.3倍**（高付加価値と説明可能な範囲） |

### こんな場合に推奨
- 業界標準の設計に乗せたい
- 稟議の通しやすさ最優先
- 顧客に「分かりやすい」を最重視

## パターンC：パッケージ＋追加席型

### プラン

```
┌──────────────────────────────────────────────────┐
│ FREE          │ PERSONAL      │ TEAM            │
│ 無料          │ ¥9,000 /月    │ ¥49,000 /月〜   │
│               │               │                 │
│ 月100実行     │ 1名           │ 5席込み         │
│ 1名           │ 無制限実行    │ +席 ¥9,900/席  │
│ ワークフロー3個│ 組織共有なし  │ 組織テンプレ共有│
└──────────────────────────────────────────────────┘
         例：5名¥49,000／10名¥98,500／20名¥197,500
```

### 狙い
現行の価格レンジ（1人あたり1万円前後）を維持しつつ、「5名〜」という下限表記を「5席込み＋追加席」という加算的表現に転換する。支払総額はほぼ同じだが、稟議上の見え方と心理ハードルが大きく変わる。

### メリット
- 現行価格感を踏襲（粗利モデル変更最小）
- 人数拡大でスケールする（大規模顧客もENTERPRISE不要）
- 「5席込み」＝パッケージ商品の匂い、下限表記回避
- 稟議の表題が「5名契約」→「チームプラン導入」に変わる

### デメリット
- Yoom Businessよりは高価格（10名で¥98,500 vs Yoom ¥15,000）
- 「席」という概念が残るので、純粋な定額制ほどスッキリしない
- 追加席の管理プロセスが必要（契約途中の席追加UX）

### Yoom比較
| | Kawaru TEAM | Yoom Business |
|---|---|---|
| 10名月額 | ¥98,500 | ¥15,000 |
| 価格差 | Yoom比**約6.5倍**（パターンAよりは縮小） |

### こんな場合に推奨
- 現行価格レンジを大きく変えたくない
- 大規模顧客も自然にスケールさせたい
- 「席」の概念は残してOK

## パターンD：ワークスペース軸型（Jinba型）

### プラン

```
┌─────────────────────────────────────────────────┐
│ FREE          │ PERSONAL      │ TEAM            │
│ 無料          │ ¥9,000 /月    │ ¥29,800 /月     │
│               │               │                 │
│ 1ワークスペース│ 1ワークスペース│ 3ワークスペース │
│ 1名           │ 1名           │ 人数無制限      │
│ 月100実行     │ 無制限実行    │ 無制限実行      │
│ 組織共有なし  │ 組織共有なし  │ 組織テンプレ共有│
└─────────────────────────────────────────────────┘
```

### 狙い
人数軸を完全に捨て、「ワークスペース数」で組織らしさを表現する。1ワークスペース＝1部署／1プロジェクトなどの単位で区切る。

### メリット
- 人数問題（下限／上限／1人あたり）から完全離脱
- 部署単位・プロジェクト単位の導入ストーリーが作れる
- Jinbaの設計を参考にでき、差別化の独自軸を持てる
- 人数無制限で社内展開の摩擦ゼロ

### デメリット
- 「ワークスペース」概念を顧客に説明する必要あり（認知負荷）
- 機能設計としてWSが既に存在するか開発確認が必要
- 単価がパターンA～Cより低くなる傾向
- 日本市場で「ワークスペース課金」の慣習が浅く、説明コストが高い

### Yoom比較
| | Kawaru TEAM | Yoom Business |
|---|---|---|
| 月額 | ¥29,800 | ¥15,000 |
| 軸 | WS数 | タスク数 |
| 価格差 | Yoom比**約2倍** |

### こんな場合に推奨
- 独自軸で他社と差別化したい
- 将来的に部署別・プロジェクト別の切り売りを視野
- プロダクト側にワークスペース概念が既にある or 実装予定

## 🔍 選び方ガイド

### 優先順位別の推奨

| 大川PMが最優先するもの | 推奨パターン |
|-----------------------|-----------|
| 既存設計の踏襲・早期ローンチ | **A** |
| 顧客の分かりやすさ・稟議通りやすさ | **B** |
| 現行価格レンジの維持＋表現改善 | **C** |
| 独自差別化・新概念導入 | **D** |

### 判断フロー

```
Q1. TEAM の 1人あたり計算（¥14,900／人）を外せるか？
  ├─ YES → Q2へ
  └─ NO  → パターンA

Q2. 価格レンジを下げてもOK（顧客拡大重視）？
  ├─ YES → パターンB
  └─ NO  → Q3へ

Q3. 「ワークスペース」という新概念を導入する開発余地ある？
  ├─ YES → パターンD
  └─ NO  → パターンC
```

## 🌟 推奨：パターンC または B

### 推奨理由

**パターンC（パッケージ＋追加席型）を第一推奨**：

1. 大川PMの「9,000円／14,900円は相場感の仮置き」を最大限尊重しつつ、「5名〜」下限表記の問題を解消できる
2. 「5席込み」と「追加席」は日本市場で違和感なく受容される表現（ボーダフォンのシェアプランや保険商品で慣例化）
3. 支払総額は現行案とほぼ同じ（5名チームは両方¥74,500 vs ¥49,000だが、大型は揃ってくる）
4. β版でテストしやすい設計（追加席課金をオフにすれば実質パターンBに近づく）

**パターンBは次点**：
- 業界トレンドとの整合性を最優先するなら
- 10名以内のSMB（中小企業）を中心顧客に絞るなら

### 共通の追加設計（どのパターンを選んでも推奨）

1. **14日TRIAL → 無期限FREEに変更**
   - 業界標準（34社中28社が無期限Free提供）
   - 月100実行／ワークフロー3個のような厳しい機能制限で「試す壁」だけ下げる
2. **「β版価格は期間限定」を明示**
   - 正式版ローンチ時（例：2026年12月）に改定予定と予告
   - 既存契約者への移行設計を事前告知
3. **税込表示併記**
   - 「¥9,000（税抜）／¥9,900（税込）」の両表記
   - LP一瞥時の印象と契約時金額の乖離を防ぐ
