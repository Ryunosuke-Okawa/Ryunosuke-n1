## エグゼクティブサマリー

Yoom株式会社は、**日本国内の業務自動化（ハイパーオートメーション）市場を牽引するSaaS企業**。創業者がAIの大波を確信し、2019年に設立したTimeTechnologiesから、2022年に独立法人として切り出した。700以上のSaaS連携・AI機能・RPAを1つのノーコードプラットフォームに統合することで、Zapierなどグローバル競合が攻略できていない「日本の中小企業市場」に深く食い込んでいる。

## 2. 創業の経緯・事業化の背景

### 前身：株式会社TimeTechnologies

2019年1月、波戸﨑駿が**株式会社TimeTechnologies**を設立。当初はAI技術の台頭を確信し、「時間を生み出す」をテーマに起業。同社では2事業を並行展開していた。

| 事業 | 内容 | その後 |
|------|------|--------|
| AutoLine | LINE特化型マーケティングオートメーション | 2022年に**株式会社ブレインパッドへM&A売却** |
| Yoom | ビジネスプロセス自動化（BPA）ツール | 2022年に独立法人として切り出し |

### 法人化の経緯：スピンアウトではなく「カーブアウト」

一般的な「スピンアウト（親会社から独立）」ではなく、**自社の2事業のうち1つを売却して資金確保し、もう1つに全集中する形で独立法人化**した。ビジネス的には「カーブアウト（Carve-out）」に近い意思決定。

| 時系列 | 出来事 |
|--------|--------|
| 2019年1月 | TimeTechnologies設立 |
| 2021年7月 | Yoomクローズドβ版リリース |
| 2022年 | AutoLine事業をブレインパッドへ売却（選択と集中） |
| 2022年6月 | Yoom株式会社を新会社として設立 |
| 2022年7月 | TimeTechnologiesからYoomへ事業譲渡 |
| 2022年8月 | シリーズA 8.4億円調達 |
| 2024年4月 | 事業承継完全完了（IP・顧客・人員すべて移行） |

## 3. プロダクト：Yoomとは何か

### ハイパーオートメーションとは（Gartner定義）

> 「AI・ML・RPA・iPaaS・OCRなど複数の自動化技術を組み合わせ、業務プロセス全体をエンドツーエンドで自動化すること」（Gartner、2020年提唱）

単なる「1ツールによる自動化（RPA）」ではなく、複数技術の統合による**業務フロー全体の自動化**がハイパーオートメーション。Yoomはこれを1プラットフォームで提供している点が独自性。

### Yoomの技術構成

```
【Yoomの4技術統合】

AI（生成AI）         → テキスト解析・要約・感情分析・AIワーカー
API連携（iPaaS）     → 700以上のSaaSとノーコードで接続
RPA                  → APIのないレガシーシステムのブラウザ操作を自動化
OCR                  → 紙書類・PDFのデータ化

→ これら4つが1プラットフォームに統合されているのがハイパーオートメーション
```

### 競合との比較

| 比較項目 | Yoom | Zapier | Make |
|---------|------|--------|------|
| 開発国 | 日本 | アメリカ | チェコ |
| 連携数 | 700+ | 7,000+ | 1,900+ |
| 国産SaaS連携 | **非常に充実** | 限定的 | 限定的 |
| AI機能 | AIワーカー（自律型） | 限定的 | 限定的 |
| RPA/OCR | あり | なし | なし |
| UI | 日本語・ノーコード | 英語主導 | 視覚的だが難解 |

**Zapierに対する差別化ポイント**: 連携数では劣るが、国産SaaS（kintone・SmartHR・freee等）への深い対応と、AIワーカーによる「データを流すだけでなく、AIが判断して実行する」能力でポジション差別化。

## 4. ビジネスモデル

### 料金体系（PLG型）

| プラン | 月額（税別） | 対象 |
|--------|------------|------|
| フリー | 0円 | 個人・試用 |
| ミニ | 9,600円〜 | 特定部署の定型業務 |
| チーム | 24,000円〜 | 組織的な管理・DB共有 |
| サクセス | 48,000円〜 | 大規模連携・導入支援付き |

### PLG（Product-Led Growth）戦略

```
現場担当者がフリープランで使い始める
        ↓
便利さが社内に口コミで広がる（非エンジニアが80%以上）
        ↓
チーム・部署単位での有料プランへアップグレード
        ↓
全社導入・サクセスプランへ
```

従来のエンタープライズiPaaSが月額数十万円だったのに対し、月額9,600円〜という**価格破壊**により中小企業市場を一気に開拓。

## 5. 販売戦略・チャネル

### 直販（PLG）と間接販売の2本柱

#### 間接販売パートナーの構造

| パートナー種別 | 具体例 | 役割 |
|--------------|--------|------|
| DXコンサル会社 | MOVED、JCS（ジャパンコンピューターサービス）、StartLink | 中小企業への導入支援・販売代行 |
| 業務SaaS連携 | **サイボウズ（kintone）オフィシャルパートナー** | kintoneユーザー3万社以上へのリーチ |
| 出張・経費管理 | ピカパカ | 相互連携＋顧客紹介 |
| エンタープライズ系 | NTTデータビジネスブレインズ | API連携（ClimberCloud） |

#### 地方銀行・商工会議所との連携（詳細不明）

社内調査資料に「**地方銀行や商工会議所を通じた販路拡大**」との記載があるが、**具体的な銀行名・団体名の公式発表はなし**。

ただし、経済産業省・日本銀行が地方銀行・商工会議所を「中小企業DX支援の推進機関」として公式に位置づけており（2023〜2024年）、国策と完全に一致した販路構造。

```
経産省・日銀の政策→地銀・商工会議所が中小企業DX支援を強化
                        ↓
       具体的なツールとしてYoomのような国産ノーコードSaaSを紹介
                        ↓
    Yoom側：大量の中小企業へのリーチ
    地銀側：取引先へのDX支援実績が積める
```

#### サイボウズパートナー認定の意義

2022年9月にサイボウズ（kintone）の「オフィシャルパートナー」認定を取得。kintoneのソリューション紹介サイトにYoomが掲載されることで、kintoneユーザー企業からの双方向の送客が発生する構造。

## 6. 開発体制：アジャイル開発×少数精鋭

### エンジニア14名で700連携を実現する仕組み

**結論：標準化されたコネクタフレームワーク＋Custom App（ユーザー自作連携）の組み合わせ**

```
①コネクタ量産フレームワーク（自社開発）
  → 一度フレームワークを作ると、新しいSaaSのAPIを
    テンプレートに流し込むだけで追加可能
  → アジャイル開発で毎日新連携を高速リリース

②Custom App機能
  → APIドキュメントを読めるユーザーが自分でコネクタを設定
  → 700連携のうち一定数はユーザー自作のカスタム連携が含まれる

③外部ミドルウェアの活用
  → CData Connect Serverなど、1接続で複数データソースに対応できる
    中間ツールも部分的に活用
```

### unified.toのような統合APIプラットフォームは使っているか？

**使っていない可能性が高い**

unified.to・Merge.dev・Nangoなどの「Unified API」プラットフォームは存在するが、Yoomには不向きな理由が3つある。

1. **対象SaaSが違う**: 西洋のUnified APIプラットフォームはSalesforce・Workday・BambooHR等グローバルSaaSが中心。kintone・SmartHR・freeeなど**日本国産SaaSはほぼ未収録**
2. **二重構造の問題**: Yoom自身がiPaaSなので、別のiPaaSの上に乗るのは冗長
3. **日本語対応**: 日本企業のAPIドキュメントは日本語が多く、国内エンジニアチームが直接開発するほうが速い

### エンジニア規模の効率比較

| サービス | 連携数 | エンジニア規模 | 効率（連携/人） |
|---------|--------|--------------|---------------|
| Zapier | 7,000+ | 500名以上（全社） | 約14 |
| **Yoom** | **700+** | **14名** | **約50** |

Zapierと比較してYoomは1人あたり約3.5倍の効率。コネクタフレームワークの標準化の成果と推察される。なお、すべての連携が深く実装されているわけではなく、基本的なRead/Write操作のみ対応の連携も含まれると推定。

## 8. カスタマーサポート体制

### 使用ツール：Intercom（インターコム）

ヘルプセンターがIntercomのURLになっており、Intercomを採用していることが確認できる。Intercomは**AI自動回答＋人的チャット対応を両立**できるCS定番ツール。

ユーザーが体験するサポートの構造：

```
FAQ・汎用的な操作方法 → IntercomのAIが自動回答
企業固有のフロー設計・
複雑なトラブル         → CS担当者が人的対応
```

### プランによるサポート差

| プラン | サポート内容 |
|--------|------------|
| フリー・ミニ | チャット・メール対応 |
| チーム・サクセス | チャット＋メール＋Webミーティング（月2回） |
| サクセス以上 | 専門スタッフによる導入支援付帯 |

## 10. 今後の戦略

| 戦略軸 | 内容 |
|--------|------|
| AI-Native化 | AIを機能の1つではなくプラットフォームの前提に。業務データを学習してAIが自動提案 |
