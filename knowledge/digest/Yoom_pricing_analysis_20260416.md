# Yoom 2026年5月 料金改定の背景分析

## エグゼクティブサマリー

Yoomの料金改定は「単なる値上げ」ではなく、**iPaaS（フローボットの会社）→ AIエージェントプラットフォームへの事業転換**に伴う料金体系の再設計。SaaS業界全体のper-seat pricing脱却トレンドとも一致しており、4つの課題を同時に解決する構造になっている。

---

## 改定概要

### プラン体系の変更
- Before：フリー / ミニ / スタンダード / プロ / チーム / サクセス（6プラン）
- After：フリー / パーソナル / ミニ / チーム / サクセス（5プラン）
- スタンダード・プロは廃止

### 料金変更（税別・年間契約）

| プラン | Before | After | 変動 |
|--------|--------|-------|------|
| フリー | 0円 | 0円 | ― |
| パーソナル | ― | 2,400円 | 新設 |
| ミニ | 9,600円 | 16,000円 | +67% |
| チーム | 24,000円 | 40,000円 | +67% |
| サクセス | 48,000円 | 80,000円 | +67% |

### 主な変更点
- ライセンス課金の廃止（ミニ3名→20名、チーム10名→100名、サクセス20名→無制限）
- 無料タスク数の拡張（ミニ3,000→5,000、チーム10,000→15,000、サクセス30,000→45,000）
- フローボット上限の拡張（ミニプラン中心）
- 限定機能の開放（上位プラン限定だった機能がパーソナル・ミニでも利用可能に）
- サポートの強化（チームプランでWeb会議サポート付与）

## Yoomの機能進化タイムライン

```
2021年8月  β版リリース（TimeTechnologies時代）— 基本のiPaaS「フローボット」
2022年1月  正式版リリース（連携アプリ約100）
2022年6月  Yoom株式会社設立、8.4億円調達（導入約800社）
2023年9月  ★ AIオペレーション（β版）リリース（導入4,500社超）
2024年3月  Yoomナレッジ（ChatGPT活用）リリース
2024〜25年 連携アプリを100→300→700+に急拡大（導入2万社超）
2026年2月  ★★ AIワーカー機能リリース ← 大転換
2026年5月  ★★★ 新料金プラン改定
```

## Yoomの3層構造

### 第1層：フローボット（2021年〜）= 日本版Zapier
- 「トリガー」と「オペレーション」を繋いで自動化
- 人間が設計した通りに動く。判断はしない
- Zapierの「Zap」、Makeの「Scenario」と同等

### 第2層：AIオペレーション（2023年〜）= フローの中でAIを使う
- フローボットの1ステップとしてAI機能を差し込める
- OCR、データ抽出、音声文字起こし、要約、翻訳、テキスト生成
- 使用AI：Claude、ChatGPT、Groq等を選択可能
- まだ「フローボットの部品」。フロー全体の設計は人間

### 第3層：AIワーカー（2026年2月〜）= 自律型AIエージェント
- 「フローを人が設計する」のではなく「AIが自分で考えて動く」
- 依頼の意図・ニュアンス・文脈を理解して必要なステップを自律実行
- 700+のアプリを自分で操作可能
- 全プラン（フリー含む）で利用可能

## 背景分析：4つの仮説

### 仮説① PLGファネルの断崖を埋める
- フリー（0円）→ ミニ（9,600円）の間に巨大な断崖があり、個人ユーザーが離脱していた
- ITreviewで「9,600円は個人には高い」という声が複数
- パーソナルプラン（2,400円）で中間ステップを設置 → PLG転換率の改善を狙う
- PLGの基本：個人が無料で試す → 有料化 → チームに共有 → 組織導入

### 仮説② 収益構造の転換（単純な値上げではない）
- 旧モデル：低い基本料金 × ユーザー数課金 → 利用拡大に比例してコスト増
- 新モデル：高い基本料金 × ユーザー無制限（枠内）→ 社内展開の摩擦ゼロ
- SaaS業界全体でper-seat pricing脱却が加速（2025年：per-seat採用率57%、前年64%から低下）
- Zapierもチームプラン$69/月でユーザー数無制限に移行済み

### 仮説③ プラン選択の一軸化
- 旧：人数・フローボット数・オペレーション数・機能制限・タスク数の多軸判断
- 新：基本的に「何人で使うか」の一軸
  - 1人：パーソナル / 〜20人：ミニ / 〜100人：チーム / 無制限：サクセス
- Bain & Company：「シンプルな料金体系はそれ自体が競争優位」

### 仮説④ AIワーカーのマネタイズ＋普及（真の触媒）
- スライド2枚目：「AIワーカーを中心に、より便利なサービスをご提供していくことを目的として」
- AIワーカーリリース（2月）→ 料金改定（5月）のタイミングが極めて近い
- 基本料金67%アップの一部はAIコンピュート費用（LLM API代）の転嫁
- 全プランにAIワーカーを開放して「Yoom = AIで自動化」のブランド確立

### 総合
| 課題 | 解決策 |
|------|--------|
| PLGファネルの断崖 | パーソナルプラン（2,400円）で中間ステップ |
| ライセンス課金が展開ブレーキ | ライセンス廃止＋無料枠大幅拡大 |
| プラン選びの複雑さ | 「何人で使うか」の一軸に整理 |
| AIワーカーのマネタイズ＋普及 | 全プラン開放＋基本料金に転嫁 |

## SaaS業界のトレンドとの一致

### per-seat pricing脱却の動き
- per-seat採用率：64%（2024）→ 57%（2025）に低下
- usage-based pricing採用率：45%（2021）→ 61%（2025）に上昇
- 2026年Q1「SaaSpocalypse」— AIエージェントが人の仕事を代替する時代、「人数×単価」モデルは構造的リスク

### 主要事例
- **Zapier**：Team $69/月でユーザー無制限。課金軸はタスク数
- **Intercom**：AI Agent「Fin」に成果課金（1解決あたり$0.99）
- **HubSpot**：View-Only Seatを無料・無制限に
- **Adobe**：Generative Creditへ移行

### 競合比較
| | Yoom | Zapier | Make |
|---|---|---|---|
| 基本自動化 | フローボット | Zap | Scenario |
| フロー内AI | AIオペレーション | AI Actions | AI modules |
| 自律型エージェント | AIワーカー（2026/2） | Zapier Agents（2024〜） | Maia |
| 日本SaaS連携 | ◎（700+国産SaaS） | △ | △ |
| AIワーカー無料利用 | ◎（フリープランでも5体） | ✕（有料のみ） | ― |

## ユーザー実態

- ITreview満足度4.4/5
- 導入2万社超、中小企業・スタートアップが中心
- 主な不満：タスク実行数の制限、旧最安9,600円は個人に高い
- フリープラン（月100タスク）→ 有料プランへの断崖が最大の離脱ポイントだった
- パーソナルプラン（2,400円）への反応は好意的

---

## Kawaru事業への示唆

1. PLGの「フリー→有料」の中間ステップは重要。断崖があると離脱する
2. per-seat pricingはAI時代に構造的リスク。タスク/利用量/成果ベースの課金軸を検討すべき
3. 料金のシンプルさ自体が差別化要因
4. AI機能を全プランに開放することでブランドを確立する戦略は有効
