## 1. HelloPay サービス概要（髙橋氏より）

### 立ち上げ経緯
- HelloPay運営グループでは既にGMOペイメント／ソニーペイメントなど大手決済代行を使い相当の流通があったが、**手数料が高いという課題**から自前の決済代行を内製化。
- 自社で原価仕入れし、抜き幅を抑える＝**既存決済からの切替インセンティブを作る**思想で開始。

### 強み
- **業界最安水準の手数料率**（業種・リスク区分による）。
  - ハイリスク商材の場合: 4.5% 程度。
  - 一般商材は減価ベースで個別提示（返金率や他社決済コストをヒアリングして見積）。
- インフラ既存稼働、1,000億規模を支える基盤あり。
- 初期費用・月額費用は**基本的に後出しじゃんけん（=他社条件を見て合わせる）**。

### プラン構成
- 一般業種: 月商400万 / 月商2,000万以下で区切り。
- 他業種（マッチング／オンライン講座／工学／コーチング等）は事業単位で個別決済プラン。

### 対応決済
- カード: VISA / Mastercard / JCB / AMEX
- モバイル決済: Apple Pay / Google Pay
- コード不要の決済リンク・トークン決済 一通り対応
- 売上管理ダッシュボード、Webhook通知、入金管理
- 入金サイクルは一般商材で**月2回**（他PSPと同水準）
- サブスク設定、不正検知、チャージ枠管理 完備

### セキュリティ・実績
- クレカ協会の最新セキュリティ基準に準拠。フレーム型SDK実装済。
- グループ会社で実装済の例:
  - ゴージャス（マッチングアプリ）
  - オートリザーブ（予約管理）
  - ヘルスケア系サブスク（恵比寿のカロ社、年間流通5億規模） など
- 営業は基本紹介経由、広告は使っていない。

## 2. エヌイチ側の現状ヒアリング

### サービス構成（吉野氏より）
- 事業は大きく **個人向け** と **法人向け** の2軸。
  - 法人向け: 4プラン程度（最廉価はSaaSツール、上位はサポート・研修・経営コンサル付き）
  - 個人向け: 5〜6サービス。直近で「ChatGPT道場」など複数立ち上げ。
- 提供形態: 動画教材＋チャットサポート＋Zoom面談。
- サブスク/一括の比率は **法人最廉価プランがカードサブスク**、上位は一括銀行振込が中心。

### 現在の決済構成
- **メインはUNIVAPAY（旧Univapay）**: 手数料率約**3.6%**、トランザクション費用なし、月額なし、ローリングリザーブなし。利用者側に手数料転嫁。
  - 4月度カード決済流通: **約2,700万円**
- サブ: **MOSH（モッシュ）**
  - 4月度: 約50万円
- その他（審判系少額）: 数%
- 比率感: UNIVAPAY 約82% / CBS 約10% / その他少額

### 現在の課題
1. **新サービスを立ち上げるとUNIVAPAYで新規アカウントを取得できない**状況。
   - → 新サービス向けに新しい決済代行を開拓する必要あり。
2. **過去に審査落ち2〜3社**（フィンコード等で保留）。
   - 経緯: 2025年7月にユーザーからのクレーム発生 → 弁護士経由で「契約書の読み合わせ未実施」「広告表現が強め」を指摘 → 同年12月に**事業者リストに登録され通りにくくなった**。
3. **口座振替の決済手段が未導入**。
   - サブスクプランで「カードを持たない／使えない」顧客向けに口座振替を新たに入れたい。

## 3. 決済切替・導入可否のすり合わせ

### HelloPay対応可否
| 項目 | 状況 |
|------|------|
| カード決済（VISA/Master/JCB/Amex） | ✅ 対応 |
| サブスク決済 | ✅ 対応 |
| **口座振替** | ⚠️ 現状未対応。今後設定予定があるか髙橋氏が社内確認 |
| 審査リスト登録時の通過可能性 | アクワイアラ側で音社のサービスが登録されているか / 落ちた理由を髙橋氏が確認。**基本「プランは通したい」スタンス** |

### サポート体制
- 他社と同等のサポート水準を維持。安い代わりにサービスが落ちることはない。
- 操作で困った場合はメール・電話・LINE等で即時対応。
- HelloPay代表がエンジニア出身のため**技術問い合わせも即レス対応可能**。

### 手数料感
- UNIVAPAYの3.6%水準より低くできる可能性高。
- ただし業種区分と返金率次第。具体個別見積はサービスごとに必要。

## 所感・論点メモ（大川追記）

- **吉野さんが商談を引き受けたが、サービス内容と既存決済の数値把握が手元になく即答できない場面が多かった**。次回までに以下を整理しておくと交渉が進む:
  - サービス別の流通額・サブスク/一括比率
  - 過去の審査落ちの正式な経緯資料（弁護士指摘内容、広告表現の論点整理）
  - 口座振替が必要な対象プラン（どのプランで、想定何件か）
- HelloPay側は **「基本プランは通したい」「後出しじゃんけんで条件合わせる」** スタンスで前向き。条件交渉余地は大きい。
- UNIVAPAYで新規アカウントが取れない事情がボトルネック。**新サービス向け決済を別系統で確保する戦略**として、HelloPayは有力候補。
- 口座振替対応可否が今回最大の論点。HelloPay側で導入予定があるなら一気に提携メリットが大きくなる。
