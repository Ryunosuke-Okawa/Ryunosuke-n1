# write-article — 執筆チームによるコンテンツ・提案書作成

3レビュアー（髙橋／CFO／顧客目線）＋ライターの4エージェントが連携して、高品質なコンテンツまたは提案書を生成するスキル。

## トリガー

「記事書いて」「コンテンツ作って」「チームで提案書作って」「/write-article」

## ワークフロー

```
[入力: テーマ・目的・対象読者]
        ↓
[フェーズ1: 並列レビュー]
  ├── 髙橋レビュアー（戦略・本質・So What?）
  ├── CFOレビュアー（数字・整合性）         ← 同時並列実行
  └── 顧客目線レビュアー（受け手の感じ方）
        ↓
[フェーズ2: 統合・構成設計]
  └── メインClaudeが3視点を統合し、構成案＋執筆指示書を作成
        ↓
[フェーズ3: 執筆]
  └── ライター（3レビュアーの指摘を統合した原稿）
        ↓
[フェーズ4: 最終レビュー]
  └── 該当レビュアーが最終原稿をチェック
       （提案書 → 髙橋＋顧客目線、数字含む → CFO追加）
        ↓
[出力: output/articles/ または marketing/ へ保存]
```

## 実行手順

### 準備（ヒアリング）

以下が未提供の場合は質問する：
- **テーマ**：何について書くか
- **目的**：提案書 / ブログ記事 / SNSコンテンツ / LP原稿 / 社内資料
- **対象読者**：誰が読むか（クライアント業種・役職など）
- **ゴール**：読んだ後に読者に何をしてほしいか
- **文字数・形式**：指定がある場合

### フェーズ1：並列レビュー（テーマ・素材に対する事前レビュー）

`Agent` ツールで3レビュアーを**同時並列**起動する。

各レビュアーが見る視点：
- **髙橋レビュアー**：戦略の筋・本質的な問い・「で、何が言いたい？」
- **CFOレビュアー**：数字が絡む場合のみ起動。整合性・期ズレ・税表記
- **顧客目線レビュアー**：受け手として何が刺さるか・何が引っかかるか

> 数字が絡まないコンテンツ（記事・コラム等）は CFOレビュアーをスキップしても良い。

### フェーズ2：統合・構成設計

メインClaudeが3レビューを統合し：
- 構成案（H2・H3レベル）
- ライターへの執筆指示書（トーン・強調点・避けるべき言葉）

を作成する。

### フェーズ3：ライター執筆

`writer.md` のエージェントが最終原稿を執筆。
3レビュアーの指摘を文章内で消化し、痕跡を残さない。

### フェーズ4：最終レビュー

該当レビュアーが最終原稿をチェック：
- 合格（修正不要）→ そのままファイル保存
- 修正必要 → ライターに差し戻し、修正後再提出

## 保存先

- 提案書：`output/articles/proposal_[相手名]_[YYYYMMDD].md`
- 記事：`output/articles/article_[タイトル略称]_[YYYYMMDD].md`
- LP原稿：`marketing/[用途]_lp_copy_[YYYYMMDD].md`

## 育成サイクル

成果物が完成したら、各エージェントの「学習ログ」に：
- 良かった指摘
- 外した指摘
- 行動原則・判断基準への反映

を追記する（大川さんのフィードバックを起点に）。
