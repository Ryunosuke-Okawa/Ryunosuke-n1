# write-article — 執筆チームによるコンテンツ・提案書作成

リサーチャー3人・編集長・ライターの5エージェントが連携して、高品質なコンテンツまたは提案書を生成するスキル。

## トリガー

「記事書いて」「コンテンツ作って」「チームで提案書作って」「/write-article」

## ワークフロー

```
[入力: テーマ・目的・対象読者]
        ↓
[フェーズ1: 並列リサーチ]
  ├── リサーチャーA（論考分析）
  ├── リサーチャーB（事実調査）  ← 同時並列実行
  └── リサーチャーC（批判的思考）
        ↓
[フェーズ2: 統合・構成設計]
  └── 編集長（3つのリサーチを統合し構成案を設計）
        ↓
[フェーズ3: 編集長レビュー]
  └── 編集長（4軸評価・ライターへの指示）
        ↓
[フェーズ4: 執筆]
  └── ライター（フィードバックを反映した最終原稿）
        ↓
[フェーズ5: 最終レビュー]
  └── 編集長（最終チェック・承認 or 修正指示）
        ↓
[出力: output/articles/ へ保存]
```

## 実行手順

### 準備（ヒアリング）

以下が未提供の場合は質問する：
- **テーマ**: 何について書くか
- **目的**: 提案書 / ブログ記事 / SNSコンテンツ / 社内資料
- **対象読者**: 誰が読むか（クライアント業種・役職など）
- **ゴール**: 読んだ後に読者に何をしてほしいか
- **文字数・形式**: 指定がある場合

### フェーズ1：並列リサーチ

`Task` ツールを使い、リサーチャーA・B・Cを**同時並列**で起動する。

各リサーチャーの指示書:
- `researcher-a.md` — 論考分析（テーゼ・アンチテーゼ・ジンテーゼ）
- `researcher-b.md` — 事実ベース調査（データ・事例・トレンド）
- `researcher-c.md` — 批判的思考（弱点・リスク・反論の洗い出し）

### フェーズ2：編集長による統合・構成設計

3つのリサーチ結果を `editor-in-chief.md` に渡し、構成案を作成する。

### フェーズ3：編集長レビュー

構成案を4軸（洞察の深さ / 論理の明快さ / So What? / 構成の完成度）で評価。
ライターへの具体的な執筆指示を作成する。

### フェーズ4：ライター執筆

`writer.md` のエージェントが最終原稿を執筆。

### フェーズ5：最終レビュー

編集長が最終原稿をレビューし：
- 合格（85点以上）: そのままファイル保存
- 修正必要（85点未満）: ライターに差し戻し、修正後再提出

## 保存先

- 提案書: `output/articles/proposal_[相手名]_[YYYYMMDD].md`
- 記事: `output/articles/article_[タイトル略称]_[YYYYMMDD].md`
