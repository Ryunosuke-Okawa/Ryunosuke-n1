# daily-schedule — 朝の工程表生成

毎朝の工程表を作成するスキル。以下の手順で実行する。

## 実行手順

0. **ヒアリング（必須）**
   - スキル起動直後に以下を必ずユーザーに確認する（データ収集より先に実行する）：

     ```
     おはようございます！工程表を作成します。

     まず確認させてください：
     📝 今日やりたいこと・頭の中にあるタスクがあれば教えてください。
     （NAシートやSlackにない、自分だけが把握しているものも含めて）

     ない場合はそのままEnterで進めます。
     ```

   - ユーザーから入力があった場合、そのタスクを「頭の中タスク」として記録し、以降のステップで収集したタスクと同等に扱う
   - タスクが箇条書きや文章で来た場合も、個別タスクに分解して取り込む

1. **祝日判定（スキップ判定）**
   - `mcp__google-calendar__list-events` で今日の予定を以下の2ソースから取得する：
     - カレンダーID: `ja.japanese#holiday@group.v.calendar.google.com`（日本の祝日カレンダー）
     - カレンダーID: `primary`（自分のカレンダー）
   - 以下のいずれかに該当する場合、**工程表生成をスキップ**し「今日は祝日のためスキップします 🎌」と返す：
     - 「日本の祝日」カレンダーに当日の終日予定がある
     - primaryカレンダーに「祝日」を含む予定（「祝日ブロック」「祝日」等）がある
   - 該当しない場合、次のステップに進む

2. **スケジュール設定の読み込み**
   - `/Users/kyouyuu/claude/00_context/memories/preferences.md` を読み込み、工程表のルールを確認する
   - 以下のデフォルト値を設定し、preferences.mdの内容で上書きする：
     - 稼働時間帯: **10:00〜19:00**
     - 昼休憩: **30分**（MTGの合間などに柔軟に配置）
     - バッファ時間: なし
   - このルールを以降の全ステップで遵守する

1. **Slackタスクチャンネルから未完了タスクを抽出**
   - `mcp__slack__slack_get_channel_history` でチャンネルID `C0ALWNKAXMJ`（大川タスク管理）の直近100件を取得する
   - 以下を**除外**する：
     - `subtype` があるシステムメッセージ（channel_join など）
     - `reactions` に `white_check_mark`（✅）が含まれるメッセージ → 完了済み
   - 残ったメッセージをすべて未完了タスクとして工程表に組み込む

2. **Google Calendarから今日の予定を取得**
   - `mcp__google-calendar__list-events` で今日（00:00〜23:59）の予定をすべて取得する
   - 連携済みカレンダーすべてを対象にする
   - 終日予定・ミーティング・締め切りを把握し、スケジュール組みに反映する

3. **Notionからタスク・TODOを取得**
   - `mcp__cee31924-5b98-444b-bf8e-cbc8f49c894d__notion-search` で以下のキーワードで検索し、タスク・TODO管理DBを特定する：
     - 検索キーワード例：「タスク」「TODO」「今日」「期限」
   - 取得したDBまたはページから**未完了**かつ**今日〜直近が期限**のタスクを抽出する
   - ステータスが「完了」「Done」のものは除外する
   - 取得できたタスクは「Notionタスク」として工程表に組み込む

4. **Google Sheetsからタスク・案件情報を取得**
   - `mcp__google-workspace__searchDrive` で「タスク」「案件」「プロジェクト」などのキーワードでスプレッドシートを検索する
   - 見つかったシートのうち最も関連性が高いものを `mcp__google-workspace__readSpreadsheet` で読み込む
   - 未完了タスク・直近の締め切り・アクションアイテムを抽出して工程表に反映する
   - 複数のシートが見つかった場合は、更新日時が最近のものを優先する

5. **ローカルファイルのタスク確認**
   - `DAILY.md` を読み込み、昨日の未完了タスクを確認する
   - `03_projects/` フォルダの各プロジェクトフォルダを確認し、期限の近いタスクを把握する

6. **タスクの統合・分類**
   Slack・Google Calendar・Notion・Google Sheets・ローカルファイルから収集したタスクを統合し、重複を排除した上で以下の2軸で分類する：
   - **緊急度 × 重要度**（アイゼンハワーマトリクス）
   - **認知負荷**（高 / 中 / 低）

7. **工程表の生成**
   - 15分刻みで1日のスケジュールを組む
   - **稼働時間は10:00〜19:00**（10時より前、19時以降には予定を入れない）
   - **昼休憩30分を必ず入れる**（MTGの合間など、柔軟に配置してよい）
   - 高認知負荷タスクは午前中（MTG前）に優先配置
   - 打ち合わせ・タスクの後にバッファ時間は設けない（終了後すぐ次の予定を入れてよい）

   **所要時間の見積もり基準（タスク種別ごとのデフォルト値）:**
   - 架電・電話連絡: 30分（1件あたり。複数件は件数×15〜20分で）
   - メール作成・返信: 20〜30分
   - 資料確認・修正（軽微）: 30分
   - 資料作成・提案書作成: 90〜120分
   - 契約書確認: 45分
   - 打ち合わせ・商談: カレンダーの実際の時間を使う
   - NAシートの整理・タスク確認: 15分
   - 朝のレビュー・優先順位確認: 15分
   - その他不明なタスク: 30分（短く見積もらない）

   ※ 上記はあくまでデフォルト。タスク名から明らかに長くかかると判断できる場合は延長する。「〜確認」「〜チェック」は最低30分を確保する。

   - 形式:
     ```
     10:00-10:15  朝のレビュー・優先順位確認
     10:15-11:30  [タスク名] ← 高集中作業
     ...
     12:30-13:00  昼休憩
     ...
     ```

8. **DAILY.mdへの書き込み**
   - 本日の工程表を `DAILY.md` の「今日の工程表」セクションに書き込む
   - 日付を更新する

9. **Google Calendarへの書き込み**
   - カレンダーに**まだ登録されていないタスク・作業ブロック**のみを追加する（既存のMTGは追加しない）
   - `mcp__google-calendar__create-event` を使用してイベントを作成する
   - カレンダーID: `primary`、タイムゾーン: `Asia/Tokyo`
   - **色分けルール**:
     - 社内会議（全体定例・チーム定例など）: `colorId: "5"` （黄色・Banana）
     - 社外打ち合わせ・商談・会食: `colorId: "4"` （ピンク・Flamingo）
     - タスク・集中作業ブロック: `colorId: "8"` （濃い青・Blueberry）
   - 基本的にAIが新規追加するのはタスク（青）のみ
   - 既にカレンダーに登録済みの予定は重複して追加しない

10. **SlackへのDM送信**
   - 生成した工程表を Slack DM で本人（ユーザーID: `U076GASNT26`）に送信する
   - `mcp__slack__slack_post_message` を使用し、`channel_id: "U076GASNT26"` に投稿する
   - メッセージ形式:
     ```
     おはようございます！ {日付}の工程表です。

     {工程表（テキスト形式）}

     今日もよい1日を！
     ```
