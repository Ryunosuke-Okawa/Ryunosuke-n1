# weekly-report — 週次レポート生成

週次の業務・成果をまとめたレポートを生成するスキル。

## トリガー

「週次レポート作って」「今週の振り返り」「週報作成」

## 実行手順

1. **対象週の確認**
   今週（月〜日）を対象とする。別の週を指定された場合はその週を対象にする。

2. **情報の収集**
   以下のソースから情報を収集する：
   - `DAILY.md` の今週分のエントリ（完了タスク・未完了タスク・メモ）
   - `03_projects/` の各プロジェクトフォルダの更新状況
   - `00_context/memories/decisions.md` の今週の意思決定

3. **レポートの構成**
   以下の形式で生成する：

   ```markdown
   # 週次レポート — YYYY年MM月DD日（月）〜MM月DD日（日）

   ## 今週のハイライト
   （最も重要な成果・進捗を3つ以内で箇条書き）

   ## 完了したこと
   ### クライアント案件
   - [案件名]: （進捗・完了事項）

   ### 内部業務
   - （内部タスクの完了事項）

   ## 進行中・持ち越し
   - [タスク名]: 現在の状況、次のアクション

   ## 今週の学び・気づき
   -

   ## 来週の優先タスク
   1.
   2.
   3.

   ## KPI・指標（把握できる範囲で）
   -
   ```

4. **ファイル保存**
   - ファイル名: `output/digest/weekly_[YYYYMMDD].md`
   - チャットにも表示する
