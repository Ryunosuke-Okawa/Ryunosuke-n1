# inbox-organize — Gmail 受信トレイ一括整理

受信トレイのメールを読んで分類し、ラベル付け＆アーカイブする。

## トリガー

「**受信トレイ整理**」「**メール整理**」「**Inbox整理**」「**受信箱整理**」

---

## ラベル定義

| ラベル名 | Label ID | 対象 |
|----------|----------|------|
| クライアント | `Label_3784966997852734996` | 取引先・顧客・商談先との往来 |
| 社内 | `Label_2377459123798032020` | 社内メール・社内ツール通知 |
| 営業メール | `Label_8220910026580841552` | ウェビナー案内・BDR営業・外部マーケ |
| その他 | `Label_8674766807905158525` | システム通知・自動送信・スクリプトエラー等 |

---

## 実行手順

### STEP 1: 受信トレイのメール一覧取得

`mcp__google-workspace__listGmailMessages`（account: `main`）で取得：

```
labelIds: ["INBOX"]
maxResults: 50
```

---

### STEP 2: メール内容を並行読み込み

`mcp__google-workspace__readGmailMessage` を **10件ずつ並行**で実行（format: `metadata`, maxLength: 400）。

取得する情報：
- From（差出人ドメイン・氏名）
- Subject（件名）
- To / Cc（宛先・CC）
- snippet（本文冒頭）

---

### STEP 3: 分類ルール

#### 【クライアント】
以下のいずれかに該当：
- 取引先・顧客のドメインからのメール（n1-inc.co.jp 以外の会社メール）
- 件名に案件名・社名が含まれ、商談・契約・提案・納品に関する内容
- 自分（r.okawa）または 髙橋（y.takahashi）がTo/Ccに入っており、社外の人物が絡んでいる

例：中西製作所、MIL、日本事務器、Leadinxなど具体的な取引先

#### 【社内】
以下のいずれかに該当：
- From が `@n1-inc.co.jp` ドメイン
- Backlog通知（`@backlog.com`）
- ジョブカン通知（`@donuts.ne.jp`）
- Slack週次更新（`@slack.com`）
- Figma通知（`@email.figma.com`）
- Google Calendar招待で社内メンバー（n1-inc.co.jp）発

#### 【営業メール】
以下のいずれかに該当：
- ウェビナー・セミナー・カンファレンスの案内
- BDR（外部企業からのテンプレ的な営業・提案）メール
- メルマガ的な情報発信（サービス案内、事例紹介等）

#### 【その他】
上記のどれにも当てはまらない：
- Google Apps Script エラー通知
- Googleセキュリティ通知、Microsoftアカウント通知
- メール配信失敗通知（mailer-daemon）
- Stripe / Anthropic / Manus / SearchApi 等ツールのシステムメール
- HENNGE認証コード等の自動送信

#### 分類に迷う場合
- 差出人が社外かつ件名に具体的な案件名がない → **営業メール**
- 社内メンバーがToに入っているが内容が社外案件 → **クライアント**

---

### STEP 4: ラベル付け（batchAddGmailLabels）

分類ごとに **`mcp__google-workspace__batchAddGmailLabels`** を**並行**で実行：

```
クライアント → labelIds: ["Label_3784966997852734996"]
社内         → labelIds: ["Label_2377459123798032020"]
営業メール   → labelIds: ["Label_8220910026580841552"]
その他       → labelIds: ["Label_8674766807905158525"]
```

---

### STEP 5: 返信要否の判定（アーカイブ前）

以下に該当するメールは **INBOXに残す**（アーカイブしない）：

- 先方から具体的な質問・依頼・確認が来ている
- 商談・打ち合わせのアレンジや日程調整の返答を求められている
- 資料・情報の送付依頼がある
- クライアントからの返信待ちが発生しそうな内容

以下は**アーカイブ対象**（返信不要）：
- 自動送信メール（noreply、notification 等）
- 一方通行の情報共有・お知らせ
- システム通知・ツール通知
- 営業メール・ウェビナー案内（返信しない前提）

**判定に迷う場合はINBOXに残す（安全側に倒す）。**

---

### STEP 6: INBOXからアーカイブ（batchRemoveGmailLabels）

返信不要と判断したメールのみ、受信トレイから除外：

```
mcp__google-workspace__batchRemoveGmailLabels
labelIds: ["INBOX"]
messageIds: [返信不要と判断したメールのIDのみ]
```

**要返信メールはINBOXに残したままラベルだけ付与する。**

---

### STEP 7: 完了報告

以下の形式で出力する：

```
✅ 受信トレイ整理完了（YYYY-MM-DD）

| ラベル       | 件数 | 主な差出人/内容 |
|------------|------|--------------|
| クライアント |  N件 | [会社名など]  |
| 社内        |  N件 | [ツール名など] |
| 営業メール  |  N件 | [内容メモ]   |
| その他      |  N件 | [内容メモ]   |

残り受信トレイ: 約N件（続けて整理しますか？）
```

残りのメールがある場合（next page token が返ってきた場合）、ユーザーに続行確認する。

---

## 注意事項

- 未読の重要そうなクライアントメールが含まれていた場合、アーカイブ前にコメントする
- 既にラベルがついているメールは、ラベルを上書き追加する（既存ラベルは消さない）
- `INBOX` ラベルの除去 = アーカイブ。削除ではないのでGmail上でラベルフォルダから確認できる
