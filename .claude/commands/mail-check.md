# mail-check — Gmail＆Slack 未返信チェック＆返信下書き生成

GmailとSlackを横断して未返信・未スタンプのメッセージをピックアップし、返信文を自動生成するスキル。

## トリガー

「**Gメール確認**」「**メール確認**」「**メール見て**」「**返信すべきメールある？**」「**Slack確認**」「**連絡確認**」

---

## 実行手順

GmailとSlackを**並行して**チェックし、まとめて結果を出力する。

---

## 【A】Gmail チェック

### A-1: 未返信メールの取得

`mcp__google-workspace__searchGmail`（account: `main`）で以下の条件で検索する：

```
query: "newer_than:7d -from:me -label:sent (is:unread OR is:read)"
maxResults: 30
```

取得したメールを `mcp__google-workspace__readGmailMessage` で読み込み、以下を把握する：
- 差出人（会社名・氏名・メールアドレス）
- 件名・本文の要旨
- 受信日時
- スレッドID・メッセージID

**対象期間: 直近1週間（7日以内）。最大30件**

---

### A-2: 返信要否の判定

各メールを以下の基準で分類する：

#### 【返信が明らかに必要】
- 先方から具体的な質問・依頼・確認が来ている
- 商談・打ち合わせのアレンジが必要
- 資料・情報の送付依頼がある
- 日程調整の返答を求められている

#### 【返信が不要 / 除外】
- 自動送信メール（noreply、no-reply、notification、alert 等）
- メルマガ・広告・サービス通知
- 自分が送信したメール
- すでに既読で内容がお知らせ系・一方通行の情報共有

#### 【返信すべきか不明】
- 上記のどちらとも判断できないもの

## 【B】Slack チェック

大川さんのSlackユーザーID: `U076GASNT26`

### B-1: DMチャンネルの確認

`mcp__slack__slack_list_channels`でDM（タイプ: `im`）チャンネル一覧を取得し、各DMの直近履歴を確認する。

`mcp__slack__slack_get_channel_history` で各DMチャンネルの直近**1週間分**を取得（最大30件）。

**未返信DM判定ルール:**
- 最後のメッセージが自分（`U076GASNT26`）以外のユーザーから来ている
- かつ、そのメッセージに自分からの返信（`mcp__slack__slack_get_thread_replies` でスレッドを確認）がない
- かつ、自分がそのメッセージにリアクション（スタンプ）をつけていない

```
// reactions 配列に自分のuser_idが含まれているかチェック
// reactions: [{ name: "white_check_mark", users: ["U076GASNT26", ...] }]
```

### B-2: メンション（@大川）の確認

`mcp__slack__slack_list_channels` でパブリック/プライベートチャンネル一覧を取得。

各チャンネルの履歴から `<@U076GASNT26>` を含むメッセージを検索（直近1週間分）。

**未返信メンション判定ルール:**
- メンションを含むメッセージに、自分からのスレッド返信がない
- かつ、自分がそのメッセージにリアクションをつけていない

### B-3: 未返信・未スタンプの判定基準

| 状態 | 判定 |
|------|------|
| 相手の最終メッセージ → 自分が返信済み | ✅ 対応済み（スキップ） |
| 相手の最終メッセージ → 自分がスタンプ済み | ✅ 対応済み（スキップ） |
| 相手の最終メッセージ → 返信もスタンプもなし | ❗ 要対応 |
| メンション → 自分が返信 or スタンプあり | ✅ 対応済み（スキップ） |
| メンション → 返信もスタンプもなし | ❗ 要対応 |

## 【D】返信文の生成ルール

### Gmail 返信文ルール

**MEMORY.mdのメールルールを厳守する。**

#### 冒頭フォーマット
- 宛先行: `[会社名（正式名称）]\n[相手の名前] 様`
- 挨拶: `お世話になっております。\n株式会社エヌイチの大川です。`

#### 会社名の正式名称チェック
- 前株（株式会社〇〇）か後株（〇〇株式会社）かを正確に使う
- 不明な場合は WebSearch で確認してから使用する

#### 本文の方針
- 先方の質問・依頼に明確に答える
- 確認が必要な事項は「確認の上ご連絡いたします」と記載
- 次のアクションを明記する

#### 署名（変更しないこと）
```
―――――――――――――――――――――――――――――――
株式会社エヌイチ
Kawaru事業部
大川 龍之介（Okawa Ryunosuke）

TEL：080-2646-2420
Mail：r.okawa@n1-inc.co.jp
Website：https://n1-inc.co.jp/

「AI（アイ）ある社会に 」
―――――――――――――――――――――――――――――――
```

#### スレッド引用
- 本文・署名の下に元メールを引用形式（`>`）で含める

---

### Slack 返信文ルール

- 社内メンバーへの返信はカジュアルで簡潔に（敬語不要）
- 社外相手が含まれるチャンネルはフォーマルに
- スレッド返信が基本（`mcp__slack__slack_reply_to_thread` を使用）
- 承認系・確認系の場合は返信の代わりにリアクション（✅ `white_check_mark`）でも可
- 了解・受け取り確認のみの場合は 👀 (`eyes`) リアクションを提案

## 【E】送信・保存の実行（ユーザー承認後）

### Gmail下書き保存

「保存して」「これで下書き作って」と指示があった場合のみ実行。

google-workspace MCPツールは日本語件名を文字化けさせるため、**PythonでGmail APIを直接呼び出す**：

```python
# 認証: ~/.google-mcp/tokens/main.json から読み込み
# email.header.Header(subject, 'utf-8') で件名エンコード
# In-Reply-To, References に元メッセージのMessage-IDを設定
# threadId を指定してスレッド返信に紐づけ
# base64.urlsafe_b64encode → users().drafts().create()
```

保存後: 宛先・件名・スレッド紐づけ状態を報告し「Gmailの下書きフォルダからご確認・送信をお願いします」と案内。

### Slack返信・スタンプ送信

「送信して」と指示があった場合:
- `mcp__slack__slack_reply_to_thread` でスレッドに返信

「スタンプだけ」と指示があった場合:
- `mcp__slack__slack_add_reaction` で該当メッセージにリアクション追加
  - 了解・確認: `eyes`（👀）
  - 完了・承認: `white_check_mark`（✅）
