# write-proposal — クライアント提案書・資料作成

クライアント向けの提案書・提案資料を自動生成するスキル。

## トリガー

「提案書作って」「資料作って」「プロポーザル作成」「〇〇の提案を作って」

## 実行手順

1. **情報収集（ヒアリング）**
   以下の情報が未提供の場合は質問する：
   - クライアント名・担当者名
   - 提案するサービス・テーマ（例：経営戦略コンサルティング、組織改革支援）
   - 課題・ニーズ（クライアントが抱えている問題）
   - 提案の期間・規模感（概算）
   - 特に強調したい点・差別化ポイント

2. **既存情報の参照**
   - `00_context/memories/clients.md` でクライアント情報を確認
   - `03_projects/` で既存案件情報を参照
   - `01_strategy/` で事業方針・強みを確認

3. **提案書の構成設計**
   以下の標準構成で作成する（変更がある場合は指定）：
   ```
   1. エグゼクティブサマリー
   2. 課題認識（クライアントの現状と課題）
   3. 提案の概要（アプローチ・方法論）
   4. 期待される成果・効果
   5. 実施スケジュール
   6. 費用・条件
   7. 私たちについて（実績・強み）
   8. 次のステップ
   ```

4. **ドラフト生成**
   - Markdown形式でドラフトを作成
   - 具体的な数値・事例は`[要確認]`プレースホルダーで示す

5. **ファイル保存**
   - ファイル名: `output/articles/proposal_[クライアント名]_[YYYYMMDD].md`
   - チャットにも全文を表示する

## 出力形式

Markdownドラフトをチャットに表示 → `output/articles/` に保存
