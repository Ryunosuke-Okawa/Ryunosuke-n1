# internal-meeting-minutes — 社内議事録生成

社内会議の文字起こしテキストから、参加者自動特定・議事録生成・NAシート転記・Notion保存を一括実行するスキル。

## トリガー

「**社内議事録**」のみ

## 入力ファイルの取得（起動時に必ず実行）

以下の優先順位で入力を取得する：

### ① ユーザーが直接提供した場合（スキップ）

以下のケースはファイル検出をスキップし、提供されたテキストをそのまま使用する：
- チャットにテキストを直接貼り付けた場合
- ファイルパスを指定した場合

### ② テキスト未提供の場合 → Google Drive Notta フォルダから自動検出

**MCPツール**: `mcp__google-workspace__listFolderContents`（account: `main`）
**フォルダID**: `1qS5ayYNas2Xw1v1VP24LdDqu-7X7Wjgy`

フォルダ内のファイル一覧を取得し、ユーザーに選択させる：

```
📄 Google Drive「Notta」フォルダから文字起こしファイルを検出しました：
1. 3_23 幹部MTG.txt（66 KB、15:47更新）
2. 3_23 SNSユニット定例.txt（83 KB、15:47更新）
どのファイルを使いますか？（番号で選択）
```

選択が確定したら、`mcp__google-workspace__downloadFromDrive`（account: `main`）でダウンロードし、Readツールで内容を読み込む。

**ファイル命名パターン（Notta）:**
- `M_D [会議名].txt`（例: `3_19 幹部MTG.txt`）
- `会議名そのまま.txt`（例: `SNSユニット定例.txt`）

ファイル名に `M_D` パターン（例: `3_23`）がある場合は会議日付として使用する。ない場合はファイルの更新日時またはファイル内容から日付を推測する。

## 実行手順

---

### STEP 0: 参加者の特定

ファイル取得後、CLAUDE.mdの「メールアドレス→氏名・所属マッピング」および「ニックネーム→正式名変換」を使い、文字起こし内の発言者名を正式名に変換する。

不明な参加者がいる場合は、NAシートの「担当者マスタ」シート（スプレッドシートID: `1Ll5LsGL99n0P35msUqa23UgBvm9xgkE4Ke0Fn5Dxztw`）を参照。それでも不明な場合はユーザーに確認する。

特定した情報をユーザーに確認する：
```
📋 会議情報を特定しました：
- 会議名: [ファイル名 or 内容から推測]
- 日時: YYYY-MM-DD HH:MM〜HH:MM
- 参加者: [名前1]、[名前2]、...
この情報で議事録を作成してよいですか？
```

---

### STEP 1: 議事録生成

文字起こしテキストを解析し、社内会議の議事録を作成する。

#### 前処理ルール

CLAUDE.mdに定義された以下のルールを必ず適用する：
- **ニックネーム→正式名変換**: 「ばつねじ」→住友駿仁、「日向」→小林日向 など
- **社内用語補正**: 「リスティング数」→「リストイン数」、「パネル」→「ファネル」、「川村」→「河村」など
- 文字起こし特有の冗長表現（「えっと」「あの」等）は除去し、発言の意図を正確に要約する

#### 発言者の特定

文字起こしテキストに発言者ラベルがある場合（例: `大川龍之介:` `speaker_0:` 等）：
- STEP 0で特定した参加者リストと照合して正式名に変換する
- `speaker_0` 等の匿名ラベルの場合、発言内容・文脈・声の特徴的な表現から参加者を推測する
- 推測に自信がない場合はユーザーに確認する

発言者ラベルがない場合：
- 文脈から誰の発言かを推測し、各トピックの議論で「誰が何を言ったか」を整理する

#### 議事録フォーマット（社内会議用）

```markdown
# [会議名] 議事録
日時: YYYY-MM-DD HH:MM〜HH:MM
場所: オンライン（Zoom） / オフライン
参加者: [氏名リスト（所属ユニット付き）]
議事録担当: 大川龍之介

## アジェンダ
（会議の流れから主要トピックを抽出し、番号付きリストで記載）

## 議論・決定事項

### [トピック1]
- 背景:（なぜこの議題が挙がったか）
- 議論:（誰がどのような意見を述べたか。発言者名を明記）
- 決定:（何が決まったか。決まっていない場合は「継続検討」等）

### [トピック2]
...

## ネクストアクション
| # | タスク | 担当者 | 期日 |
|---|--------|--------|------|
（会議中に明示的に割り振られたタスクのみ記載。担当者は正式名で）

## 次回MTG
- 日時:（決まっていれば記載）
- 議題:（決まっていれば記載）

## 備考
（補足情報、共有事項、注意点など）
```

**ネクストアクションの担当者特定ルール:**
- 会議中に「〇〇さんお願い」「〇〇やっておいて」等の発言があればその人を担当者にする
- 「これやります」等の自発的な発言はその発言者を担当者にする
- 担当が不明確な場合は「要確認」と記載し、ユーザーに確認する
- 期日も同様に、会話から推測。不明なら次回MTGまで or 1週間後

### STEP 2: NAシート転記

議事録のネクストアクションをGoogle SpreadsheetのNAシートに転記する。

- **スプレッドシートID**: `1Ll5LsGL99n0P35msUqa23UgBvm9xgkE4Ke0Fn5Dxztw`
- **シート名**: `新規事業部`
- **MCPツール**: `mcp__google-workspace__appendSpreadsheetRows`（account: `main`）
- **フォーマット**:
  - A列: タスク名（会議名を含めて具体的に記述。例:「【幹部MTG】〇〇の件、△△に連絡する」）
  - B列: 期日（YYYY/MM/DD形式）
  - C列: 担当者（正式名）
  - D列: 進行状況 → 常に「未着手」
  - E列: 備考（どの会議で決まったかを簡潔に）

転記前にユーザーに内容を確認し、承認を得てから実行する。

---

### STEP 3: Notion転記

社内MTG DBに該当ページを検索し、議事録を保存する。

**DB情報**:
- DB URL: `https://www.notion.so/1f5934e4905a80c6b1f5c093e17026bf`
- Data Source: `collection://1f5934e4-905a-80ae-bfb6-000b68a6e2fd`
- スキーマ: MTGタイトル（title）、開催日（date）、参加者（person）、ステータス（status）、責任者（person）

#### 3-1. 既存ページの検索（必ず指定DataSource内で確認）

**検索は必ず `data_source_url: collection://1f5934e4-905a-80ae-bfb6-000b68a6e2fd` を指定して行う。**

`notion-search` で会議名＋日付を検索する：

```json
{
  "query": "[会議名] [YYYY-MM-DD]",
  "data_source_url": "collection://1f5934e4-905a-80ae-bfb6-000b68a6e2fd",
  "filters": {
    "created_date_range": {
      "start_date": "YYYY-MM-DD",
      "end_date": "YYYY-MM-DD"
    }
  }
}
```

ヒットしたページを `notion-fetch` で取得し、**開催日プロパティ（`date:開催日:start`）と会議名が一致するか**を確認する。

- 一致するページが見つかった → **3-2へ（既存ページに追記）**
- 該当日のページが見つからない → **3-3へ（新規作成）**

#### 3-2. 既存ページがある場合

該当ページに議事録コンテンツを追加する（`notion-update-page`）。
ページ末尾に議事録本文をMarkdown形式で追記する。

#### 3-3. 既存ページがない場合（新規作成）

該当日のページが存在しない場合のみ、`notion-create-pages` でデフォルト空ページとして新規作成する。
作成時に**プロパティ・議事録本文をすべて一括で設定する**（後から手動入力は行わないため）：

- **親DB**: `https://www.notion.so/1f5934e4905a80c6b1f5c093e17026bf`
- **プロパティ（全項目を設定）**:
  - `MTGタイトル`: [会議名]（例: 「幹部MTG」「SNSユニット定例」）
  - `date:開催日:start`: YYYY-MM-DD
  - `ステータス`: 「完了」
  - `参加者`: STEP 0で特定した参加者（Notionユーザーが特定できる場合）
  - `責任者`: 大川龍之介
- **コンテンツ**: 議事録本文（Markdownフォーマット）を全文設定

#### 3-4. 完了報告

Notionページ作成 or 更新後、URLをユーザーに報告する。

## エラーハンドリング

- **カレンダーに該当する会議がない場合**: ユーザーに会議名・参加者を直接聞く
- **参加者のメールアドレスが不明な場合**: 文字起こしの発言者名とCLAUDE.mdのマッピングから推測。推測できない場合はユーザーに確認
- **NAシートのMCPツールが使えない場合**: NAの内容をテキストで表示し、手動転記を依頼

## 完了報告

全ステップ完了後、以下のサマリーを表示する：

```

3. NAシート転記 → [N]件転記済み（新規事業部シート）
4. Notion保存 → [新規作成 or 既存ページに追記]（URL: ...）
```
